The removed TypeScript-owned no-throw calls are:

```ts

```

No new Go operation, sync-specific Go handler, native helper rebuild, or new native smoke key is introduced.

## Runtime Behavior

`clearPaneLabelSync(paneId): void` remains synchronous, public no-throw, and returns no value. Pane ids are compactly validated as a `%123`-style tmux pane id before the helper call.

The TypeScript facade now delegates through the synchronous kernel adapter:

```ts
export function clearPaneLabelSync(paneId: string): void {
  try {
    createAgentTeamKernelAdapter().clearPaneLabel(paneId)
  } catch (_) {}
}
```

The hidden direct TypeScript fallback for this behavior is removed: `tmux/panes.ts` no longer calls the direct `runTmuxNoThrow(...)` clear-label command pair and no longer imports `runTmuxNoThrow` for this helper.

Invalid pane id, helper missing, tmux missing, command failure, timeout, and incompatible response failures must not throw publicly from `clearPaneLabelSync(...)`. The adapter returns compact failure results internally and the public facade swallows failures to preserve the historical no-throw/void cleanup API.

## Reused Go Operation

The synchronous adapter method reuses existing `workerLifecycle.clearPaneLabel`:

- `createAgentTeamKernelAdapter().clearPaneLabel(paneId)` validates `paneId` compactly as a `%123`-style tmux pane id.
- It calls `callHelper<unknown>('workerLifecycle', { operation: 'clearPaneLabel', paneId: requestedPaneId })`.
- It validates the helper response with the existing `validatePaneLabelClearingResult(...)` path.
- It uses existing compact `workerLifecycleUnavailablePaneLabelClearing(...)` diagnostics and compact `recordRuntimeFallback(...)` detail for incompatible responses.

The only authorized Go tmux command surface remains the existing v0.6.78 clear-label command pair:

- `tmux set-option -up -t <paneId> @agentteam-name`
- `tmux select-pane -t <paneId> -T ''`

These commands remain argv-only. Raw pane input, raw tmux stdout/stderr, helper paths, stack traces, and raw helper output must not leak through public behavior, diagnostics, task reports, logs, or tests.

In compact policy terms: raw tmux stdout/stderr/helper output must not leak.
