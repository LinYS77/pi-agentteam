## Preview Mode

`go-packaged-preview` is an explicit local preview mode. It is recognized by the kernel adapter but is never selected by default.

Resolver precedence:

1. explicit helper path wins first.
   - `PI_AGENTTEAM_KERNEL_HELPER`, `AGENTTEAM_GO_KERNEL_HELPER`, or `helperPath` adapter option remains Model B local/reviewer override.
   - if an explicit helper path is present, `go-packaged-preview` uses it before any packaged preview path.
2. packaged preview helper path is considered only in `go-packaged-preview`.
   - `packagedHelperPath` adapter option or `PI_AGENTTEAM_KERNEL_PACKAGED_HELPER` is read only by `core/kernel.ts`.
   - runtime authority paths must not read packaged helper env directly.
3. packaged preview status is normalized only by the adapter seam.
   - `available` allows path discovery.
   - `unsupported-platform` maps to compact cutover failure kind `missing-helper`.
   - `integrity-failed` maps to compact cutover failure kind `helper-incompatible-response`.

The preview resolver is a skeleton for local validation. It does not add companion native packages, generated artifacts, package manifests, checksums, package installation, or normal-user availability by itself.

## Fail-Closed Behavior

In `go-packaged-preview`, `tmuxSnapshotParse` follows cutover/fail-closed behavior:

- helper unavailable, unsupported, corrupt, incompatible, non-executable, malformed, version-skewed, unsafe, timed out, crashed, or otherwise failing returns `ok:false`.
- failure snapshots include `status:"unknown"`, `resultMarker:"stale"`, module/capability `tmuxSnapshotParse`, compact `cutoverFailureKind`, empty `panes`, and empty `byPaneId`.
- the TypeScript parser fallback callback is not invoked for `tmuxSnapshotParse` failures in the preview cutover path.
- migration fallback fields `fallbackKind`, `fallbackReason`, and fallback counts are not used for preview cutover failures.
- helper output, helper paths, repo/cwd paths, mailbox/report text, raw state, sidecar/cache/index contents, worker prompts, stack traces, and package internals must not leak into compact diagnostics.

## STOP Conditions

STOP before any npm/default/native cutover if any of these are true:

- `go-packaged-preview` is treated as default or normal-user availability proof.
- package metadata changes without an owner package slice.
- native artifacts, optional native packages, lockfiles, Go modules, package scripts, lifecycle hooks, or `kernel/` package inclusion appear.
- packaged helper discovery activates outside `go-packaged-preview`.
- current `go-cutover` starts using packaged discovery without explicit follow-up approval.
- `tmuxSnapshotParse` cutover broadens to other modules.
- `compactReadModelFingerprint` becomes cutover-owned.
- preview cutover failures can silently invoke the TypeScript parser fallback.
- diagnostics leak helper path, stdout/stderr, repo/cwd paths, mailbox/report text, raw `cutoverReason`, raw state/team JSON, sidecar/cache/index, worker prompts, stack traces, or package internals.
- Go helper gains tmux execution/capture, state/repository writes, network/listeners, worker lifecycle, PlanRun/governance, full-text, package/release, or control-plane authority.
- `npm version`, `npm publish`, commit, tag, or push is proposed as part of this slice.
