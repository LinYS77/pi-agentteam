## Slice 2 — Temp NPM Clean-Install Baseline Evidence

Slice 2 adds `scripts/lib/go-helper-clean-install-proof.cjs`, `scripts/verify-go-helper-clean-install-proof.cjs`, and `tests/suites/go-kernel-v0633-package-manager-clean-install-baseline.cjs`.

The Slice 2 proof runs `npm pack <repo-root> --ignore-scripts --pack-destination <temp>` into an OS temp root, then runs `npm install <local temp tarball> --ignore-scripts --package-lock=false --legacy-peer-deps --no-audit --no-fund` in a separate OS temp project. It identifies `node_modules/pi-agentteam` as the installed package root and validates only package-installed files.

The JSON summary is redacted and carries `reviewOnly: true`, `prototype: true`, `nonAvailability: true`, `normalUserAvailability: false`, and `nativePackageDelivery: false`. Temp pack/install roots are cleaned by default.

Slice 2 remains a clean TypeScript/pi facade package install baseline only. It does not consume a verified helper, inject an installed native layout, prove real package-manager native delivery, approve install source behavior, or change default resolver/runtime behavior.

## Slice 3 — Verified Artifact to Installed Layout Consumption Evidence

Slice 3 extends `scripts/lib/go-helper-clean-install-proof.cjs` with `runInstalledLayoutConsumptionProof`, extends `scripts/verify-go-helper-clean-install-proof.cjs` with `--build-review-artifact` and `--artifact-root <path>`, and adds `tests/suites/go-kernel-v0633-installed-layout-consumption.cjs`.

The proof first runs the existing strict review artifact verifier against the review artifact. `--build-review-artifact` builds a review artifact in an OS temp root before verification; `--artifact-root <path>` consumes a caller-supplied extracted artifact root after verification.

It copies only the verified `native/tmuxSnapshotParse/0.3.0-read-model-shadow/linux-x64-glibc` layout into the temp installed package root. It loads transpiled kernel code from the temp installed package root, not from the repo source checkout.

Only explicit `go-packaged-preview` consumes the installed layout. Default/unset, `disabled`, `typescript`, `go`, and `auto` modes must not read the installed layout. `compactReadModelFingerprint` remains TypeScript fallback / non-cutover and does not call the helper.

Slice 3 is an installed-layout consumption prototype only. It is not package-manager-delivered native helper evidence, not install source approval, not normal-user availability, not default resolver approval, and not default Go approval.
