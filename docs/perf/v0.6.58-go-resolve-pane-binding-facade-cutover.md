# v0.6.58 Go resolvePaneBinding Facade Cutover

Result: v0.6.58 cuts over the TypeScript `resolvePaneBinding(paneId)` facade/default path to the Go-backed `workerLifecycle.inspectPane` adapter after extending that compact inspect result with `target`.

This slice closes the direct `display-message` fallback gap for the synchronous pane binding resolver:

```text
tmux/core.ts resolvePaneBinding(paneId) -> TypeScript kernel adapter -> Go workerLifecycle.inspectPane -> { paneId, target } | null
```

## Runtime Behavior

- Go `workerLifecycle.inspectPane` now returns compact `target` (`#{session_name}:#{window_id}`) alongside `paneId`, `currentCommand`, `inMode`, `mode`, and `copyMode`.
- `core/kernel.ts` sanitizes optional `target` on successful inspect results and preserves the existing no-text, read-only result shape.
- `tmux/core.ts` `resolvePaneBinding(paneId)` now delegates to `createAgentTeamKernelAdapter().inspectWorkerPane(paneId)`.
- The TypeScript `runTmuxNoThrow(['display-message', ...])` fallback for `resolvePaneBinding()` is removed.
- `resolvePaneBinding(paneId)` returns `{ paneId, target }` only when inspect succeeds and compact `target` is present.
- Helper failure, invalid pane id, pane-not-found, unsafe response shape, and missing target all fail closed to `null`.

## Target Field Decision

The target field belongs on the universal read-only `inspectPane` operation, not on `listAgentTeamPanes()` lookup behavior. `resolvePaneBinding()` must support arbitrary tmux pane ids, including panes without `@agentteam-name`; `listAgentTeamPanes()` remains intentionally filtered to labeled agentteam panes only.

The Go source remains read-only: it continues to use `tmux list-panes -a -F workerLifecycleInspectPaneFormat`, with the format extended to include `#{session_name}:#{window_id}`. No `display-message` or mutating tmux command is added to Go.

## Preserved Boundaries

- `resolvePaneBindingAsync()` remains TypeScript `display-message`-owned.
- `targetForPaneId()` and `captureCurrentPaneBinding()` remain TypeScript `display-message`-owned.
- `windowExists()` and `firstPaneInWindow()` remain TypeScript window helper paths.
- `listAgentTeamPanes()` remains delegated to `createAgentTeamKernelAdapter().listAgentTeamPanes()` and still filters labeled panes only.
- Mutating worker lifecycle remains TypeScript-owned: no `wakePane`, `syncPaneLabels`, `createTeammatePane`, `killPane`, `clearPaneLabel`, send-keys, label mutation, split-window, new-window, kill-pane, select-pane, or respawn-pane migration.
- State repository, task/report/PlanRun governance, team panel view-model, release/package verification, and package ownership remain unmigrated.
- Native artifact path and binary name remain unchanged.
