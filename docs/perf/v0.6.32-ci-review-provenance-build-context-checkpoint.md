## Implemented Evidence by Slice

### Slice 1 — hosted observation record validator/CLI

- `scripts/lib/go-helper-hosted-observation-record.cjs` validates a bounded local record schema.
- `scripts/verify-go-helper-hosted-observation-record.cjs` verifies a local `--record` with optional `--json` and explicit `--allow-not-observed`.
- minimal record facts are limited to schema version, approved workflow path/name, exact commit SHA, observed status, optional run id/run attempt for observed records, completed/success jobs, `linux-x64-glibc`, retention 7, and review-only/non-availability flags.
- A non-observation record uses `observed: false` and `status: "not_observed_locally"`; it must not include run id, run attempt, conclusion, or jobs.
- The validator does not use network, `gh`, tokens, hosted artifacts, downloaded bundles, raw API payloads, verifier JSON bodies, workflow summaries, or checked-in records.

### Slice 2 — provenance consistency verifier

- `scripts/lib/go-helper-artifact-verifier.cjs` adds compact `provenance-mismatch` diagnostics for cross-document consistency failures.
- The verifier checks source revision and source path consistency across `artifact-index.json`, `manifest.json`, and `provenance.json`.
- The verifier checks generatedAt, exact build command, bounded build env, cwd, toolchain, runIdentity, outputRootKind, smoke metadata, and provenance schema allowlists.
- Strict GitHub expected context keeps `manifest.build.runIdentity` and `provenance.build.runIdentity` bound to `github-run-<run_id>` and requires CI output roots to be `os-temp`.
- Diagnostics remain compact and must not leak paths, process output, URLs, stacks, raw metadata bodies, or mailbox/report text.

### Slice 3 — workflow context binding and builder consistency

- `.github/workflows/go-helper-review-artifact.yml` keeps a single build row and single verify row for `ubuntu-latest / linux-x64-glibc`.
- The review-only verifier step passes strict expected context for target, checked-out source revision, `github.sha`, `github.run_id`, `github.run_attempt`, and `github.ref`.
- `scripts/verify-go-helper-artifact.cjs` exposes `--expected-github-run-attempt` and `--expected-github-ref` for reviewer/CI verification.
- `tests/suites/go-kernel-v0632-workflow-context-binding.cjs` guards the workflow context binding, CLI/verifier mismatch behavior, no-leak diagnostics, and package/runtime boundaries.
- `tests/suites/go-kernel-v0632-builder-provenance-consistency.cjs` guards builder-generated `artifact-index.json`, `manifest.json`, and `provenance.json` consistency for source/generatedAt/target/platform/helper/protocol/package/module/capabilities/path/build command/env/cwd/toolchain/runIdentity/smoke/files/checksums/GitHub context/output-root boundaries.

### Slice 4 — final checkpoint and guard consolidation

- `docs/perf/v0.6.32-ci-review-provenance-build-context-checkpoint.md` is the final v0.6.32 provenance/build-context checkpoint.
- `tests/suites/go-kernel-v0632-ci-review-provenance-checkpoint-docs.cjs` guards route coverage, evidence files/suites, GO/STOP, tag policy, blockers, workflow shape, package/runtime guardrails, and no checked-in generated/hosted/native artifacts or raw records.
- Slice 4 is docs/tests/guard consolidation only; it does not add workflow rows or runtime/package/default behavior.
