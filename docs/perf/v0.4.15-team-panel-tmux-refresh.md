# v0.4.15 Team Panel / Tmux Refresh Baseline

> Scope: GitHub-only checkpoint prep for the Tmux Adapter + `/team` Panel Refresh Stability Gate. This is a deterministic baseline artifact, not a p95 release target pass/fail claim.

## Fixture

Attached scenario:

- Leaders: 1
- Workers: 3
- Tasks: 100
- Mailbox items: 500
- Warmup iterations: 1
- Measured iterations: 5

Global scenario:

- Teams: 10
- Workers per team: 3
- Total workers: 30
- Tasks per team: 10
- Total tasks: 100
- Mailbox items per team: 25
- Total mailbox items: 250
- Warmup iterations: 1
- Measured iterations: 5

Shared bench properties:

- Runtime/tmux: fake tmux client and fake TUI; no real tmux/pi/LLM dependency
- Full-body leak guard: fixture full mailbox/report bodies include `BENCH_PANEL_TMUX_V0415_FULL_BODY_SENTINEL_SHOULD_NOT_LEAK`; JSON output must not include it
- Profiling: `PI_AGENTTEAM_PROFILE=1` records panel `dataLoad`, `readModelBuild`, `render`, `requestRender`, `cacheHit`, `diffChanged`, plus tmux command count/duration/success/failure

## Command

```bash
npm run --silent bench:team-panel-tmux > /tmp/agentteam-v0415-team-panel-tmux-refresh-baseline.json
```

Equivalent explicit command:

```bash
PI_AGENTTEAM_PROFILE=1 node tests/bench/team-panel-tmux-refresh-v0415.cjs
```

## Output Shape

The JSON output includes:

- `fixture.attached` and `fixture.global` sizes
- `iterations.warmup` and `iterations.measured`
- `attached.panel.dataLoadMs` and `global.panel.dataLoadMs` timing percentiles
- `attached.panel.renderMs` and `global.panel.renderMs` timing percentiles
- `attached.tmux` and `global.tmux` command count/duration/success/failure summaries
- `requestRenderCount`, `cacheHitCount`, `diffChangedCount`, `customPanelCount`, and `doneCallCount`
- `implementation`, `kernel`, and `fixtureProfile` metadata for future TypeScript fallback vs optional Go-kernel parity comparisons
- `note: "baseline only; not a release target pass/fail gate"`

## Notes

- Attached warm sync/refresh remains in-place: `customPanelCount` is 1 and `doneCallCount` is 0.
- Global warm refresh uses bounded snapshot/list-panes work; this run records five profiled `list-panes` commands across five measured iterations.
- Attached warm refresh records zero tmux commands in this deterministic run because light refresh uses the warm reconcile cache and no force path is invoked.
- `cacheHitCount` records no-diff refreshes; `diffChangedCount` remains zero because the measured fixture is intentionally stable.
- No full mailbox/report body sentinel appears in the JSON output.
- `AGENTTEAM_BENCH_FIXTURE=stress` can run the larger attached/global fixture profile for scalability shape; this is still baseline/profiling data, not a release target pass/fail claim.
- Future `PI_AGENTTEAM_KERNEL=go` parity runs should compare against `PI_AGENTTEAM_KERNEL=typescript` output with matching fixture/iterations and equivalent compact output; see `docs/perf/go-kernel-parity-scaffolding.md`.
- This baseline establishes a repeatable profiling and regression comparison point. It does not claim final p95 release targets are achieved.
