# v0.6.30 CI Review Artifact Prototype

> Scope: Slice 1 introduces a controlled GitHub Actions review artifact workflow contract and guard for the existing Go helper artifact builder. This is review-only CI evidence, not package/release/default/native availability work.

## Slice 1 — CI Workflow Contract and Guard

Slice 1 adds a single review-only GitHub Actions workflow: `.github/workflows/go-helper-review-artifact.yml`.

The workflow contract is intentionally narrow:

- trigger only by `workflow_dispatch` or `pull_request` changes to the Go helper, builder scripts, or this workflow.
- permissions stay `contents: read` only.
- one required Ubuntu reviewer row sets `output_root="$RUNNER_TEMP/agentteam-go-helper-review-artifact-${{ matrix.target }}"` and builds the existing host-platform helper artifact with `node scripts/build-go-helper-artifact.cjs --output-root "$output_root" --ci-review --json`.
- output root stays under `$RUNNER_TEMP`; no repository-local artifact output is committed.
- `actions/upload-artifact@v4` uploads exactly one review-only artifact per matrix row with a name beginning `${{ matrix.artifact-name }}` and including GitHub sha/run context.
- artifact retention is limited to `retention-days: 7`.
- uploaded files are the existing builder outputs plus `artifact-index.json`: helper executable, `manifest.json`, `SHA256SUMS`, `provenance.json`, `LICENSE`, `license.json`, and placeholder-only `attestation.intoto.jsonl`.

## Explicit Non-Approvals

Slice 1 does not approve package, release, default, or runtime behavior expansion:

- no `npm version`, `npm publish`, or release asset upload.
- no `gh release`, GitHub release assets, tags, commits, or pushes.
- no `package.json` version, metadata, package files, `optionalDependencies`, package scripts, or lifecycle hooks.
- no postinstall, download, install-time build, package-manager install proof, or normal-user native availability proof.
- no `go.mod`, `go.sum`, npm lockfile, or shrinkwrap.
- no checked-in generated helper binaries, generated manifests, checksums, provenance, attestations, tarballs, or artifact bundles.
- no default Go, default resolver, current `go-cutover` behavior change, TypeScript fallback deletion, `compactReadModelFingerprint` cutover, broad Go authority, or `/team readiness` expansion.

## Slice 2 — Builder CI Mode and Artifact Index

Slice 2 extends the existing v0.6.29 builder CLI/lib with opt-in CI review index mode:

- `--artifact-index` writes `artifact-index.json` next to the generated helper metadata.
- `--ci-review` is shorthand for `--artifact-index` and is used by `.github/workflows/go-helper-review-artifact.yml`.
- `artifact-index.json` is review/transport metadata, not runtime resolver input.
- index file paths are package-relative paths only; no absolute repository, cwd, or temp paths are accepted.
- index file rows include `kind`, `path`, `sha256`, and `size` for the helper executable, `manifest.json`, `SHA256SUMS`, `provenance.json`, `LICENSE`, `license.json`, and placeholder-only `attestation.intoto.jsonl`.
- index identity fields include `schemaVersion`, `packageName`, `packageVersion`, `module`, `capability`, `helperVersion`, `protocolVersion`, `target`, `platform`, `sourceRevision`, and compact GitHub run metadata.
- missing GitHub metadata falls back to compact `unknown-*` values.
- review flags are explicit: `reviewOnly: true`, `releaseAsset: false`, `installSource: false`, and `normalUserAvailability: false`.
- retention metadata stays aligned with workflow retention: `retentionHint.days = 7` and `expiresHint = retention-days:7`.

The workflow now invokes `node scripts/build-go-helper-artifact.cjs --output-root "$output_root" --ci-review --json` and checks `artifact-index.json` exists before upload. The upload remains one review-only GitHub Actions artifact with `retention-days: 7`.

Slice 2 preserves existing generated `manifest.json`, `provenance.json`, `LICENSE`, `license.json`, `SHA256SUMS`, and placeholder-only `attestation.intoto.jsonl` schema compatibility. `core/kernelPackagedResolver.ts` and `core/kernel.ts` must not read or use `artifact-index.json`.

No artifact download/reverify verifier in Slice 2. No matrix expansion in Slice 2. No package/default/runtime resolver dependency on artifact-index in Slice 2.

## Slice 3 — First CI Matrix Row and Unsupported-Row Policy

Slice 3 introduces first-row CI matrix metadata without expanding support claims beyond one required review row:

- required CI review row is limited to `ubuntu-latest / linux-x64-glibc`.
- workflow row metadata includes `runner: ubuntu-latest`, `target: linux-x64-glibc`, `os: linux`, `arch: x64`, `libc: glibc`, `node-version: 24`, `go-version: stable`, and `required: true`.
- the workflow uses matrix values for runner, Node, Go, output root, expected artifact target path, and artifact name.
- artifact names include target/platform context plus GitHub sha/run context to disambiguate matrix rows.
- generated artifact path must match the expected matrix target before upload.
- macOS, Windows, linux-arm64, musl, and other rows remain future/experimental/unsupported until green CI smoke and separate acceptance.
- No cross-compilation without native smoke.
- GitHub Actions artifacts remain review-only: not release asset, not install source, and not normal-user availability proof.
- Slice 3 does not implement artifact download/reverify verifier.

No macOS or Windows jobs are enabled in Slice 3. Future rows may be added only as separately approved experimental/non-blocking or required rows, and must not be described as supported normal-user availability until accepted.

## Slice 4 — CI Artifact Download/Reverify Smoke

Slice 4 adds reviewer/CI transport validation for the review-only artifact bundle:

- verifier library: `scripts/lib/go-helper-artifact-verifier.cjs`.
- verifier CLI: `scripts/verify-go-helper-artifact.cjs`.
- verifier input is a downloaded artifact root plus optional package-relative artifact-index or manifest path.
- this is reviewer/CI transport validation, not runtime download, not install source, and not package-manager install proof.
- the verifier validates `artifact-index.json`, `manifest.json`, `SHA256SUMS`, `provenance.json`, `LICENSE`, `license.json`, and placeholder-only `attestation.intoto.jsonl`.
- file paths must be package-relative only; absolute paths, backslashes, and traversal are rejected.
- file `sha256` and `size` metadata must match actual extracted bytes.
- Direct JSON-RPC smoke covers `health` and `tmuxSnapshotParse`.
- Explicit preview smoke constructs `createAgentTeamKernelAdapter({ mode: 'go-packaged-preview', packagedHelperInstallRoot, packagedHelperManifestPath })` against the downloaded artifact root and proves `tmuxSnapshotParse` uses the helper instead of TypeScript fallback.
- `compactReadModelFingerprint` remains TypeScript fallback and must not call the helper in explicit preview smoke.
- compact JSON summary is emitted on success; compact no-leak JSON diagnostics are emitted on failure.

The workflow adds a dependent `verify-review-artifact` job. It uses `actions/download-artifact@v4` to download the exact review artifact uploaded by the build job into runner temp space, then runs `node scripts/verify-go-helper-artifact.cjs --artifact-root "$artifact_root" --json`.

No artifact URL/config is product/runtime input in Slice 4. No postinstall/download/install-time build is added. No release asset, npm package, package-manager install proof, default Go/default resolver, TypeScript fallback deletion, or `/team readiness` expansion is approved.

## Slice 5 — Explicit Preview Reviewer Usability Hardening

Slice 5 hardens reviewer-only explicit `go-packaged-preview` diagnostics for downloaded CI artifact root/manifest mistakes:

- manifest without root and root without manifest fail closed with compact `missing-helper` diagnostics; no default discovery is attempted.
- unsupported platform/libc, missing helper executable, version/protocol skew, checksum mismatch, and executable-mode mistakes stay compact and no-leak.
- Explicit helper path still wins over direct packaged helper path and manifest resolver; direct packaged helper path still works.
- manifest resolver failures in `go-packaged-preview` do not call the TypeScript tmux fallback.
- disabled/typescript/auto/current `go-cutover` ignore packaged manifest env and do not spawn the packaged helper.
- default/unset/go use only the approved embedded helper manifest and ignore preview fixture helper markers.
- metadata exposes only helper basename, never installed root, temp root, repository cwd, package internals, stdout/stderr, raw manifest/provenance/license/attestation/artifact-index bodies, stack traces, mailbox text, or report text.
- No `/team readiness`, ambient UI diagnostics, model-callable tools, package install, release, or broadened normal-user availability behavior is added.

The focused Slice 5 suite is `tests/suites/go-kernel-v0630-packaged-preview-reviewer-usability.cjs`.
