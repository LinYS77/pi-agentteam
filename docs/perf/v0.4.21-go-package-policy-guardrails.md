# v0.4.21 Go Package Policy Guardrails

> Scope: Slice 3 docs/tests policy only. This document freezes what remains prohibited in v0.4.21 and defines what a future package slice may intentionally change after signoff. It does not change `package.json`, add package metadata, add package scripts, add native artifacts, include `kernel/` in the package, implement a resolver, run `npm version`, run `npm publish`, approve native packaging, make Go default, or delete the TypeScript parser fallback.

## Context

Slice 1 runtime availability planning lives in `docs/perf/v0.4.21-go-runtime-availability.md`. Slice 2 native artifact contract lives in `docs/perf/v0.4.21-go-native-artifact-contract.md`.

Package behavior remains unchanged until a future Model C package implementation slice is separately approved. v0.4.21 Slice 3 only documents policy and guards.

## Frozen v0.4.21 Package Policy

The current frozen package/native policy is:

- `package.json` version remains `0.6.8`.
- `package.json#files` excludes `kernel/`.
- no lifecycle hooks: `preinstall`, `install`, `postinstall`, `prepare`, `prepublish`, `prepublishOnly`, `publish`, or `postpublish`.
- no helper build/install/download/package/version/publish scripts.
- no `package-lock.json`.
- no `npm-shrinkwrap.json`.
- no root `go.mod`.
- no root `go.sum`.
- no helper `kernel/go/agentteam-kernel/go.mod`.
- no helper `kernel/go/agentteam-kernel/go.sum`.
- no checked-in `.exe`, `.dll`, `.so`, `.dylib`, or helper artifact.
- no `optionalDependencies` for native companion packages yet.
- no native companion package metadata yet.
- no package resolver path for generated native artifacts yet.
- no package inclusion for generated helper artifacts yet.

This means v0.4.21 remains a docs/tests signoff milestone, not a native packaging implementation milestone.

## Future Package Slice Change-Control

A future package slice may intentionally change package/native behavior only after Slice 1 availability and Slice 2 artifact contract signoff are accepted.

Future slices may propose changes such as:

- `optionalDependencies` or equivalent metadata for platform-specific companion native packages.
- `package.json#files` entries for generated release artifacts or a controlled native path.
- resolver path rules for installed/generated helpers.
- checksum manifests and package/helper compatibility manifests.
- CI artifact production for supported OS/arch/libc rows.
- package dry-run checks such as `npm pack --dry-run --ignore-scripts`.
- clean install smokes from package artifacts on supported platforms.
- uninstall/upgrade cleanup checks for package-owned helper files.

Every intentional package/native change must include:

- explicit owner slice and leader approval.
- docs guard that describes the change.
- package/native sanity update that expects the new behavior.
- rollback story for bad native artifacts or bad package metadata.
- no hidden lifecycle/download/build behavior unless separately approved by a security/offline/provenance review.
- no broadening of Go authority beyond the approved `tmuxSnapshotParse` parser-only helper boundary.
- no TypeScript fallback deletion unless normal-user availability, diagnostics, rollback, and package evidence have passed a later gate.

## Prohibited-by-Default Patterns

These patterns remain prohibited by default:

- postinstall/preinstall/prepare downloads.
- install-time `go build`.
- checked-in generated binaries in the source repo.
- implicit network fetch.
- hidden bootstrap scripts.
- default Go enablement.
- TypeScript parser fallback deletion.
- package version changes in signoff-only slices.
- `npm version`.
- `npm publish`.
- helper build/install/download/package/version/publish scripts.
- lifecycle hooks.
- package lockfiles.
- root/helper `go.mod` or `go.sum`.
- `kernel/` package inclusion.
- broadening Go authority to tmux execution/capture, state/repository writes, network/listeners, worker lifecycle, PlanRun/governance, full-text boundaries, `/team` rendering, or package/release control.

## Package Sanity Checklist

Use this checklist for v0.4.21 package/native sanity:

```bash
node - <<'NODE'
const fs = require('fs')
const pkg = JSON.parse(fs.readFileSync('package.json', 'utf8'))
const lifecycle = ['preinstall','install','postinstall','prepare','prepublish','prepublishOnly','publish','postpublish']
  .filter(name => pkg.scripts && Object.prototype.hasOwnProperty.call(pkg.scripts, name))
if (pkg.version !== '0.6.8') throw new Error(`unexpected package version ${pkg.version}`)
if ((pkg.files || []).some(entry => entry === 'kernel' || entry.startsWith('kernel/') || entry.includes('/kernel/'))) throw new Error('package files include kernel')
if (lifecycle.length) throw new Error(`lifecycle scripts present: ${lifecycle.join(', ')}`)
for (const [name, command] of Object.entries(pkg.scripts || {})) {
  if (/npm\s+(?:version|publish)\b/.test(command)) throw new Error(`${name} runs npm version/publish`)
  if (/go\s+(?:build|install)\b/.test(command)) throw new Error(`${name} builds helper`)
  if (/curl\b|wget\b|node-gyp\b|prebuild/i.test(command)) throw new Error(`${name} downloads/builds native helper`)
  if (/kernel\//i.test(command) && /pack|publish|files|npm/i.test(command)) throw new Error(`${name} packages kernel/native helper`)
}
for (const file of ['package-lock.json', 'npm-shrinkwrap.json', 'go.mod', 'go.sum', 'kernel/go/agentteam-kernel/go.mod', 'kernel/go/agentteam-kernel/go.sum']) {
  if (fs.existsSync(file)) throw new Error(`${file} exists`)
}
console.log('package/native config sanity passed')
NODE

native_artifacts="$(find . -type f \( -name '*.exe' -o -name '*.dll' -o -name '*.so' -o -name '*.dylib' \) -not -path './node_modules/*' -not -path './.git/*' -print)"
if [ -n "$native_artifacts" ]; then printf '%s\n' "$native_artifacts" >&2; exit 1; fi
echo 'native artifact scan passed'
```

Checklist coverage:

- scripts scan for `npm version` or `npm publish`.
- scripts scan for `go build` or `go install`.
- scripts scan for `curl`, `wget`, `node-gyp`, or `prebuild`.
- scripts scan for `kernel/` package/publish/file operations.
- package files check excludes `kernel/`.
- lifecycle hook check rejects install/publish hooks.
- lock/go module existence check rejects `package-lock.json`, `npm-shrinkwrap.json`, root/helper `go.mod`, and root/helper `go.sum`.
- native artifact scan excludes `node_modules` and `.git` and rejects checked-in `.exe`, `.dll`, `.so`, and `.dylib` files.

## Relationship to Slices 1 and 2

- Slice 1 availability decision defines why Model B is local/reviewer-only and why normal-user availability signoff is required before npm/default/native cutover.
- Slice 2 native artifact contract defines future helper artifact identity, platform matrix, health smoke, integrity/provenance, offline/CI, cleanup, and boundary requirements.
- Slice 3 freezes current package behavior so future package implementation slices must explicitly own any package/native changes.

No future Model C package implementation may proceed until the relevant docs guard and package/native sanity expectations are updated in the slice that owns the change.

## STOP Conditions

STOP v0.4.21 package/native work if any of these appear outside an explicitly approved future package slice:

- `package.json` version changes from `0.6.8`.
- `package.json#files` includes `kernel/`.
- `optionalDependencies` or native companion package metadata is added.
- lifecycle hook is added.
- helper build/install/download/package/version/publish script is added.
- `package-lock.json` or `npm-shrinkwrap.json` appears.
- root/helper `go.mod` or `go.sum` appears.
- checked-in `.exe`, `.dll`, `.so`, `.dylib`, or helper artifact appears.
- postinstall/preinstall/prepare download appears.
- install-time `go build` appears.
- implicit network fetch appears.
- generated binaries are checked into the source repo.
- default Go enablement appears.
- TypeScript parser fallback deletion appears.
- Go authority broadens beyond parser-only stdin/stdout `tmuxSnapshotParse`.
- package/native sanity is not updated in the same owner slice as an intentional future package change.
