> Scope: Slice 1 docs/tests-only owner contract. This starts the v0.4.25 Native Helper Availability Proof Checkpoint by defining what native helper availability proof must show before default/native behavior or TypeScript fallback deletion can be discussed. It does not implement an artifact validator, runtime resolver, package metadata, native artifacts, default Go, fallback deletion, `/team readiness` expansion, or runtime behavior changes.

## T013 pi Runtime Boundary Finding

T013 pi runtime research is binding for v0.4.25:

- pi extension/provider/tool surfaces are TS/JS/Node-based.
- no native Go pi extension/provider ABI is assumed.
- TS/pi control plane remains mandatory.
- Go helper must be invoked behind TS adapter/ports via subprocess/RPC/stdin-stdout.
- Go is not a pi extension/provider surface in this plan.
- Go helper authority remains parser-only stdin/stdout `tmuxSnapshotParse` unless a later owner slice explicitly proves and approves a narrower extension.

Implication: v0.4.25 native helper availability proof is about generated helper artifacts and TS adapter/resolver behavior, not replacing the pi extension facade with Go.

## Slice 4 — Unsupported Platform and Rollback/Version-Skew Policy

Slice 4 is docs/tests policy guard work only. It defines unsupported-platform matrix, rollback/version-skew policy, and compact fail-closed diagnostics for unsafe helper/package states before any future default/native/fallback deletion can be discussed.

Focused suite: `tests/suites/go-kernel-v0425-unsupported-rollback-policy.cjs`.

Unsupported-platform matrix and remediation categories:

| Platform condition | Current policy | Remediation category |
| --- | --- | --- |
| unsupported os/arch/libc | fail closed with compact no-leak diagnostics | narrow support policy or publish supported artifact after approval |
| missing helper package/artifact | fail closed with compact no-leak diagnostics | corrected package release |
| bad package metadata | fail closed with compact no-leak diagnostics | corrected manifest/package metadata release |
| checksum/provenance/license mismatch or missing | fail closed with compact no-leak diagnostics | integrity metadata regeneration and package republish |
| non-executable helper | fail closed with compact no-leak diagnostics | corrected executable artifact release |
| stale helper | fail closed with compact no-leak diagnostics | roll forward, deprecate stale helper, or default-disable |
| helper/package version skew | fail closed with compact no-leak diagnostics | align package and helper versions |
| protocol skew | fail closed with compact no-leak diagnostics | align helper protocol with TS adapter contract |
| capability skew | fail closed with compact no-leak diagnostics | require parser-only `tmuxSnapshotParse` capability declaration |
| corrupt helper output | fail closed with compact no-leak diagnostics | reject helper and keep default TypeScript behavior |
| broken diagnostics | fail closed with compact no-leak diagnostics | diagnostics owner fixes no-leak compact UX before any default path |
| bad resolver default | fail closed with compact no-leak diagnostics | runtime owner disables/reverts default resolver decision |
| package unpublish/deprecation | fail closed with compact no-leak diagnostics | package owner publishes/deprecates corrected package policy |

Compact fail-closed cases:

- unsupported os/arch/libc → `unsupported_platform`, block default/native/fallback deletion.
- missing helper package/artifact → `artifact_missing`, block default/native/fallback deletion.
- bad package metadata → `metadata_invalid`, block default/native/fallback deletion.
- checksum/provenance/license mismatch or missing → `integrity_mismatch`, block default/native/fallback deletion.
- non-executable helper → `artifact_not_executable`, block default/native/fallback deletion.
- stale helper → `artifact_stale`, block default/native/fallback deletion.
- helper/package version skew → `version_skew`, block default/native/fallback deletion.
- protocol skew → `protocol_skew`, block default/native/fallback deletion.
- capability skew → `capability_skew`, block default/native/fallback deletion.
- corrupt helper output → `helper_smoke_failed`, block default/native/fallback deletion.
- broken diagnostics → `diagnostics_broken`, block default/native/fallback deletion.
- bad resolver default → `bad_default_resolver`, block default/native/fallback deletion.
- package unpublish/deprecation → `package_unavailable`, block default/native/fallback deletion.

Rollback scenarios and owners:

| Rollback scenario | Owner | Required rollback action |
| --- | --- | --- |
| bad metadata | package owner | corrected package metadata release and affected version deprecation |
| bad helper artifact | package owner | corrected helper artifact release and bad artifact deprecation |
| checksum/provenance mismatch | release owner | corrected release/tag/package with regenerated integrity metadata |
| unsupported platform | support-policy owner | narrow support policy or publish supported artifact after approval |
| broken diagnostics | diagnostics owner | default-disable native path until compact no-leak diagnostics are fixed |
| package unpublish/deprecation | package owner | publish/deprecate corrected package policy and default-disable affected native path |
| stale helper | release owner | roll forward, deprecate stale helper package, or default-disable native path |
| bad default resolver decision | runtime owner | corrected release/tag/package/deprecation/default-disable policy |

Rollback is corrected release/tag/package/deprecation/default-disable policy, not hidden runtime TypeScript fallback after cutover. No future cutover/default path may silently use TS parser fallback as rollback.

Unsupported platforms fail closed with compact no-leak diagnostics and keep default/native/fallback deletion blocked unless support policy is narrowed and approved.

Future normal-user diagnostics UX, if needed, must not leak helper path/stdout/stderr/repo/cwd/raw manifest/checksum/provenance/package internals/stack/mailbox/report text.
