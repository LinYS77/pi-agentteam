## Hosted Observation Record Contract

The local record validator is `scripts/verify-go-helper-hosted-observation-record.cjs`. It reads a local JSON file only; it does not query GitHub, require `gh`, require a token, download artifacts, or execute the workflow.

A minimal observed success record may include only these facts:

- `schemaVersion: 1`
- `workflowPath: ".github/workflows/go-helper-review-artifact.yml"`
- `workflowName: "Go Helper Review Artifact"`
- `commitSha`: the exact 40-hex commit under review
- `observed: true`
- `runId`: GitHub run id as a string or number
- optional `runAttempt`: GitHub run attempt as a string or number
- `status: "completed"` and `conclusion: "success"`
- `jobs`: exactly `build-review-artifact` and `verify-review-artifact`, both `completed` / `success` for `linux-x64-glibc`
- `target: "linux-x64-glibc"`
- `retentionDays: 7`
- `reviewOnly: true`, `releaseAsset: false`, `installSource: false`, `packageArtifact: false`, `normalUserAvailability: false`

A local non-observation record may include the same workflow, commit, target, retention, and flags with `observed: false` and `status: "not_observed_locally"`; it must not include `runId`, `runAttempt`, `conclusion`, or `jobs`.

## Forbidden Record Content

Do not commit hosted artifacts, downloaded bundles, verifier JSON bodies, workflow summaries, raw API payloads, `artifact-index.json`, `manifest.json`, `SHA256SUMS`, `provenance.json`, `LICENSE`, `license.json`, `attestation.intoto.jsonl`, generated manifests, generated checksums, generated provenance, generated attestations, package artifacts, native binaries, tarballs, zips, local paths, absolute paths, URLs, stdout/stderr, stack traces, mailbox/report text, or release/package/default/native availability claims.

Allowed checkpoint evidence is limited to commit, run id, run attempt, workflow status/conclusion, build/verify job result, target, retention, and explicit review-only/non-availability flags.
