## Remaining STOP Gates Before Future Default/Native Cutover

STOP future default/native cutover until all of these are satisfied in later explicit owner slices:

- real package metadata owner slice approves companion package metadata or generated artifact package inclusion.
- generated native artifacts/package contents exist for the supported platform matrix with checksum manifests, provenance/attestation, license metadata, executable-bit validation, and version compatibility checks.
- clean install smokes pass across supported OS/arch/libc matrix without Go toolchain, source checkout, manual helper env, lifecycle download, or install-time build.
- unsupported-platform policy is accepted for every AgentTeam-supported platform.
- compact diagnostics UX is implemented and no-leak guarded if a default/native parser path can fail closed in normal runtime.
- rollback story and package release process are accepted, including bad artifact/package metadata recovery.
- no hidden TypeScript parser fallback exists for default/native cutover failures.
- parser failures cannot cause pane loss, false empty snapshots, force reconcile, kill panes, worker lifecycle mutation, or stale success.
- `compactReadModelFingerprint` remains non-cutover unless separately approved after `tmuxSnapshotParse` availability is solved.
- Go helper authority remains parser-only/stdin-stdout and does not gain tmux execution/capture, state/repository writes, network/listeners, worker lifecycle, PlanRun/governance, full-text, package/release, or control-plane authority.
