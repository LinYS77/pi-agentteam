## TypeScript/pi Authority Boundary

Go remains parser-only for `tmuxSnapshotParse`.

TypeScript/pi remains authoritative for:

- pi extension loading, tools, commands, hooks, schemas, prompts, and role routing.
- tmux command execution and capture, including `list-panes` and snapshot text capture.
- pane labels, pane/window lifecycle, session lifecycle, and worker lifecycle.
- leader-gated task/message/report governance and PlanRun control.
- state/repository/sidecar/outbox writes and legacy compatibility.
- `/team` UI data loading, rendering, and compact/full-text read boundaries.

Go must not render UI. Go must not own command/tool/hook control plane. Go must not manage package/release behavior.

The only permitted future expansion from v0.6.46 is a separately authorized, non-mutating default-Go dry-run verifier for `tmuxSnapshotParse` readiness. That future dry-run still must not enable default Go or default resolver.
