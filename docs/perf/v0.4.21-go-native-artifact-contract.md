# v0.4.21 Go Native Artifact Contract

> Scope: Slice 2 docs/tests contract only. This document defines future generated helper artifact requirements before any packaging or resolver implementation. It does not change `package.json`, add package metadata, add native artifacts, implement a resolver, package the helper, change npm files, run `npm version`, run `npm publish`, approve native packaging, make Go default, or delete the TypeScript parser fallback.

## Context

Slice 1 availability planning lives in `docs/perf/v0.4.21-go-runtime-availability.md`. The v0.4.20 GitHub-only cutover checkpoint lives in `docs/perf/v0.4.20-go-cutover-checkpoint.md`.

Carry-forward decisions:

- Builds remain source-only/manual for reviewers until a future native signoff approves generated artifacts.
- `PI_AGENTTEAM_KERNEL=go-cutover` remains explicit/local-only.
- npm/default/native cutover remains STOP.
- `tmuxSnapshotParse` is the only current cutover-owned module.
- `compactReadModelFingerprint` remains TypeScript fallback / non-cutover.
- This contract is a prerequisite for future Model C package work; it is not a package implementation.

## Artifact Identity

Future helper artifacts must be named and located with explicit module/platform/version identity. A future companion package or generated artifact may use a convention like:

```text
agentteam-kernel-tmuxSnapshotParse-v0.3.0-read-model-shadow-<os>-<arch>[-<libc>][.exe]
```

A future installed package path should be module/platform/version scoped, for example:

```text
native/tmuxSnapshotParse/<helperVersion>/<os>-<arch>[-<libc>]/agentteam-kernel[.exe]
```

Contract requirements:

- artifact identity includes module `tmuxSnapshotParse`, helper version, OS, CPU architecture, and Linux libc target when applicable.
- helper belongs to `tmuxSnapshotParse` only for the current cutover path.
- artifact names must not imply Go owns `compactReadModelFingerprint`, `/team`, tmux execution, state, worker lifecycle, PlanRun, governance, full-text boundaries, or package/release control.
- `compactReadModelFingerprint` remains non-cutover TypeScript fallback and must not require a native artifact.
- generated artifact metadata must include package name/version, helper version, protocol version, module, platform tuple, checksum, provenance identifier, license metadata, and build source revision.

## Supported Platform Matrix

Future native signoff must include an explicit supported platform matrix before any helper can be resolved automatically.

Each supported row must define:

- OS, for example `linux`, `darwin`, or another explicitly approved target.
- CPU architecture, for example `x64`, `arm64`, or another explicitly approved target.
- Linux libc target when relevant, for example `glibc` or `musl`.
- executable extension and permission expectations, including `.exe` where applicable and executable bit validation on POSIX platforms.
- tmux availability assumptions and pi extension support assumptions.
- package manager behavior for optional companion packages or generated artifacts.
- CI/install smoke coverage for that platform row.

Unsupported-platform policy must be explicit:

- unsupported platforms must fail closed with compact diagnostics such as `unsupported-platform` or a documented mapping to existing failure kinds.
- unsupported platforms must not silently run the TypeScript parser fallback in a future packaged/default cutover path.
- if AgentTeam still supports a platform without a helper artifact, default/native cutover and global TypeScript fallback deletion remain STOP for that platform unless support policy is narrowed.

## Version, Protocol, and Capability Contract

Every future helper artifact must pass a JSON-RPC health compatibility check before it can be used for `tmuxSnapshotParse`.

Required health fields:

- JSON-RPC protocol version `1`.
- helper version currently `0.3.0-read-model-shadow` or an explicitly approved successor.
- capability includes `tmuxSnapshotParse`.
- `businessPathsConnected:false`.
- implementation reports Go/native helper identity without claiming TypeScript/pi control-plane authority.

Package/helper version skew must be detected:

- main package version and helper package/artifact version compatibility must be declared in a manifest.
- mismatched helper version, protocol version, module, platform tuple, or capability fails closed.
- wrong package/helper pairing must not call the helper for business logic and must not fall back silently to the TypeScript parser in a future packaged/default cutover path.

## Health and Smoke Contract

Future artifact validation must include direct helper smokes from the installed/generated artifact, independent of source checkout.

Direct `health` smoke expectations:

```json
{
  "ok": true,
  "implementation": "go",
  "protocolVersion": 1,
  "helperVersion": "0.3.0-read-model-shadow",
  "capabilities": ["health", "profile", "tmuxSnapshotParse", "tmuxSnapshotCapture", "compactReadModelFingerprint"],
  "businessPathsConnected": false
}
```

Direct `tmuxSnapshotParse` smoke expectations:

- JSON-RPC request method is `tmuxSnapshotParse`.
- input is already-captured tmux `list-panes` stdout supplied over stdin/stdout JSON-RPC.
- response preserves `capturedAt`.
- response returns `ok:true` for valid parser input.
- response returns a `panes` array and `byPaneId` map with normalized `paneId`, `target`, `label`, and `currentCommand` fields.
- response does not include mailbox/report text, raw pane text, state files, repo paths, sidecar/cache/index contents, stdout/stderr bodies, or helper paths.

Fail-closed smoke expectations:

- missing helper fails closed.
- corrupt helper fails closed.
- wrong-platform helper fails closed.
- non-executable helper fails closed.
- wrong-version helper fails closed.
- incompatible helper response fails closed.
- helper integrity failure fails closed.
- unsupported platform fails closed.
- future packaged/default cutover path must not silently invoke the TypeScript parser fallback.
- no silent TS parser fallback is allowed in any future packaged/default cutover path.
- failures return compact diagnostics compatible with `ok:false`, `status:"unknown"`, `resultMarker:"stale"`, module/capability `tmuxSnapshotParse`, stable failure kind, empty panes, and empty `byPaneId`.

## Integrity and Provenance Contract

Future native helpers must be generated release/package artifacts, not checked-in source repository binaries.

Required integrity/provenance evidence:

- checksum manifest for every artifact, with at least SHA-256 or an explicitly approved successor.
- manifest binds helper version, package version, module, OS, CPU architecture, libc target when relevant, filename, size, checksum, and build source revision.
- provenance or attestation records build workflow, source revision, toolchain identity, and artifact creation time.
- license metadata accompanies the helper artifact and any runtime dependencies.
- executable-bit validation is tested before helper use.
- corrupted, tampered, truncated, wrong-checksum, wrong-manifest, or unsigned/unattested artifact fails closed.
- generated artifacts are reproducible enough for reviewer verification or have an approved provenance explanation.
- native binaries are not committed into the source repo.

## Offline, CI, Install, and Cleanup Behavior

Future package signoff must describe how normal users and CI environments obtain the helper without source builds.

Required behavior:

- clean install on supported OS/arch can locate and execute a compatible helper without Go toolchain, source checkout, manual `/tmp` build, or `PI_AGENTTEAM_KERNEL_HELPER` override.
- offline/CI tarball or cache workflow is documented for companion packages or generated artifacts.
- install smoke verifies helper discovery, executable permissions, `health`, `tmuxSnapshotParse`, and version compatibility.
- uninstall removes package-owned helper files.
- upgrade replaces stale helper artifacts and detects package/helper version skew.
- cleanup does not remove user-provided external helpers used by Model B.
- failed install, missing optional native package, or unsupported platform produces documented fail-closed behavior and compact diagnostics.

## Boundary Constraints

The Go helper remains parser-only/stdin-stdout.

The helper must not gain:

- tmux execution or tmux capture.
- shell execution.
- state or repository reads/writes.
- package metadata writes.
- network clients, servers, or listeners.
- worker lifecycle authority.
- pane lifecycle authority.
- PlanRun, task/report governance, or leader review authority.
- mailbox/report full-text access.
- runtime `/team` rendering authority.
- package/release authority.

TypeScript/pi remains the control plane for tmux capture, `/team`, worker lifecycle, state/repository, task/report governance, PlanRun, full-text boundaries, package metadata, and release operations.

## STOP Conditions

STOP future native implementation or default/native cutover if any of these are true:

- platform matrix is vague or omits OS, CPU architecture, Linux libc target when relevant, executable extension, permission expectations, tmux assumptions, or pi support assumptions.
- unsupported-platform policy is missing.
- package/helper version skew detection is missing.
- JSON-RPC health/protocol/capability/version check is missing.
- checksum manifest, provenance/attestation, license metadata, or executable-bit validation is missing.
- helper availability relies on postinstall/preinstall/prepare download.
- helper availability relies on install-time `go build`.
- native binaries are checked into the source repo.
- lifecycle hooks, package scripts, lockfiles, `go.mod`, `go.sum`, or `kernel/` package inclusion appear before a separately approved packaging slice.
- missing/corrupt/wrong-platform/non-executable/wrong-version/incompatible helper can silently invoke the TypeScript parser fallback in a future packaged/default cutover path.
- diagnostics leak helper path, stdout/stderr, repo path, mailbox/report text, raw `cutoverReason`, raw state, sidecar/cache/index, hidden runtime state, worker prompts, stack traces, or package internals.
- Go helper authority broadens beyond parser-only stdin/stdout.
- `compactReadModelFingerprint` becomes cutover-owned without a separate approved gate.
- `package.json` version changes, `npm version` is run, `npm publish` is run, commits/tags/pushes are made, or Slice 3 starts without explicit instruction.
