# v0.6.74 Go Refresh Window Pane Labels Cutover

Result: v0.6.74 cuts over `tmux/labels.ts refreshWindowPaneLabels(target, signal)` from direct TypeScript pane-border window `set-option` calls to the Go-backed `workerLifecycle.refreshWindowPaneLabels` operation.

This implements the runtime migration authorized by the v0.6.73 gate. It is a narrow mutating tmux cutover only for window-level pane-border label refresh.

## Runtime Change

`tmux/labels.ts` keeps the explicit `windowExists(target, signal)` authority guard and then delegates to `createAgentTeamKernelAdapter().refreshWindowPaneLabelsAsync(target, signal)`.

The direct TypeScript `runTmuxNoThrowAsync(['set-option', '-w'...])` fallback for the same pane-border behavior is removed.

The public facade remains no-throw `Promise<void>`.

## Authorized Go Command Surface

Go `workerLifecycle.refreshWindowPaneLabels` may execute only these two non-destructive window-level tmux commands:

- `tmux set-option -w -t <target> pane-border-status top`
- `tmux set-option -w -t <target> pane-border-format '#{?@agentteam-name,#{@agentteam-name},#{pane_title}}'`

No other Go mutating tmux commands are introduced by this slice.

The Go implementation validates the target through the existing compact tmux window target sanitizer, uses per-call helper execution, and returns compact worker lifecycle diagnostics for invalid target, helper failure, abort, timeout, unavailable tmux, and tmux command failure.

## Public Behavior

The public facade remains no-throw `Promise<void>`.

- success resolves `void`.
- helper failure, invalid target, abort, and tmux command failure resolve without throwing at the public facade and expose only compact internal diagnostics.
- raw stdout/stderr/helper output does not leak through the public facade.
- invalid or missing target fails closed without creating sessions/windows.

## Preserved Existing Behavior

`markWindowAsAgentTeam(target, signal)` remains Go-backed with the v0.6.72 command surface.

It still uses only:

- `tmux set-option -w -t <target> automatic-rename off`
- `tmux set-option -w -t <target> allow-rename off`
- `tmux set-option -w -t <target> @agentteam-window 1`

The `markWindowAsAgentTeam` public facade still keeps `windowExists(target, signal)`, remains no-throw `Promise<void>`, and must not regain a direct TypeScript set-option fallback for the same marking behavior.

## Preserved Boundaries

pane labels, pane titles, new-session/new-window, pane creation/layout, wake/kill, state/task/UI/release/package remain TypeScript-owned.

Still out of scope for this cutover:

- pane labels via `set-option -p`
- pane titles via `select-pane -T`
- `new-session`
- `new-window`
- pane creation/split/layout/resize
- wake/send-keys
- kill/clear labels beyond existing TypeScript pane-label clear behavior
- state repository
- task/report/PlanRun governance
- team panel view-model
- release/package verification
- native artifact path or binary rename
- package manager native delivery or lifecycle hooks
