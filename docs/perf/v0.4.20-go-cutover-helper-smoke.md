# v0.4.20 Go Cutover Helper Smoke Validation

> Scope: Slice 6 source-only validation for explicit `PI_AGENTTEAM_KERNEL=go-cutover`. These commands are manual reviewer smokes only. They do not make Go default or required, delete TypeScript fallback globally, add runtime `/team` UI diagnostics, change `package.json` version, add package scripts or lifecycle hooks, run `npm version`, run `npm publish`, add package locks, add `go.mod`/`go.sum`, check in native binaries, include `kernel/` in the package, or approve native packaging.

## Purpose

v0.4.20 validates that the source-only Go helper can be built locally under `/tmp` and used explicitly for the experimental `tmuxSnapshotParse` cutover mode. This smoke complements the v0.4.19 source-only helper readiness commands in `docs/perf/v0.4.19-go-helper-smoke-readiness.md` but uses `PI_AGENTTEAM_KERNEL=go-cutover` and verifies both the helper-backed success path and the fail-closed missing-helper path.

## Source-Only Rules

- Build helper binaries only into `/tmp` with `mktemp /tmp/agentteam-v0420-kernel.XXXXXX`.
- Build with `GO111MODULE=off` from `kernel/go/agentteam-kernel`.
- Use the helper only through explicit `PI_AGENTTEAM_KERNEL=go-cutover` and `PI_AGENTTEAM_KERNEL_HELPER="$helper"` for the smoke being reviewed.
- Remove the temporary helper with `rm -f "$helper"` and clear the trap with `trap - EXIT`.
- Do not add package scripts, npm lifecycle hooks, postinstall/download steps, package locks, `go.mod`, `go.sum`, checked-in native artifacts, native packaging, npm version changes, npm publish, commits, tags, or pushes.

## Temporary Helper Smoke

Run from the repository root when the local Go toolchain is available:

```bash
helper="$(mktemp /tmp/agentteam-v0420-kernel.XXXXXX)"
trap 'rm -f "$helper"' EXIT
(cd kernel/go/agentteam-kernel && GO111MODULE=off go build -o "$helper" .)
printf '%s\n' '{"jsonrpc":"2.0","id":"snapshot-1","method":"tmuxSnapshotParse","params":{"stdout":"%1\tsession:@1\tagentteam worker\tpi\n","capturedAt":1700000000000}}' | "$helper"
PI_AGENTTEAM_KERNEL=go-cutover PI_AGENTTEAM_KERNEL_HELPER="$helper" node tests/run.cjs
PI_AGENTTEAM_KERNEL=go-cutover PI_AGENTTEAM_KERNEL_HELPER="$helper" npm run --silent bench:team-panel-tmux
rm -f "$helper"
trap - EXIT
```

Expected success signals:

- Direct helper smoke returns a valid JSON-RPC `tmuxSnapshotParse` response with `capturedAt:1700000000000`, `ok:true`, one pane `%1`, and `byPaneId["%1"]`.
- With `PI_AGENTTEAM_KERNEL=go-cutover PI_AGENTTEAM_KERNEL_HELPER="$helper"`, helper-backed `tmuxSnapshotParse` success uses the Go helper and records no migration `fallbackKind`/`fallbackReason`.
- `compactReadModelFingerprint` remains TypeScript fallback / non-cutover under `go-cutover`.
- Missing-helper or invalid-helper `go-cutover` paths return `ok:false`, `status:"unknown"`, `resultMarker:"stale"`, `module:"tmuxSnapshotParse"`, `capability:"tmuxSnapshotParse"`, and compact `cutoverFailureKind` diagnostics without invoking TypeScript parser fallback.
- Runtime `/team` UI does not expose helper paths, stdout/stderr bodies, repo paths, mailbox/report text, raw cutover diagnostics, or new user-facing diagnostics.
- The helper artifact is removed from `/tmp` after `rm -f "$helper"`.

## Missing Go Toolchain

If `go version` is unavailable, record this smoke as skipped/manual-unavailable:

```bash
if ! go version >/dev/null 2>&1; then
  echo 'optional Go cutover helper smoke skipped: local Go toolchain unavailable'
fi
```

A missing local Go toolchain does not fail the default TypeScript runtime and must not be worked around by adding package scripts, checked-in binaries, lifecycle hooks, native packaging, package locks, `go.mod`, or `go.sum`.

## Package/Native Sanity

Run after cleanup:

```bash
node - <<'NODE'
const fs = require('fs')
const pkg = JSON.parse(fs.readFileSync('package.json', 'utf8'))
const lifecycle = ['preinstall','install','postinstall','prepare','prepublish','prepublishOnly','publish','postpublish']
  .filter(name => pkg.scripts && Object.prototype.hasOwnProperty.call(pkg.scripts, name))
if (pkg.version !== '0.6.8') throw new Error(`unexpected package version ${pkg.version}`)
if ((pkg.files || []).some(entry => entry === 'kernel' || entry.startsWith('kernel/') || entry.includes('/kernel/'))) throw new Error('package files include kernel')
if (lifecycle.length) throw new Error(`lifecycle scripts present: ${lifecycle.join(', ')}`)
for (const file of ['package-lock.json', 'npm-shrinkwrap.json', 'go.mod', 'go.sum', 'kernel/go/agentteam-kernel/go.mod', 'kernel/go/agentteam-kernel/go.sum']) {
  if (fs.existsSync(file)) throw new Error(`${file} exists`)
}
console.log('package/native config sanity passed')
NODE

find . -type f \( -name '*.exe' -o -name '*.dll' -o -name '*.so' -o -name '*.dylib' \) \
  -not -path './node_modules/*' -not -path './.git/*'
```

Expected package/native signals:

- `package.json` version remains `0.6.8`.
- `package.json#files` excludes `kernel/`.
- no lifecycle or publish hooks exist.
- no helper build/download/package script exists.
- no `package-lock.json`, `npm-shrinkwrap.json`, root/helper `go.mod`, root/helper `go.sum`, checked-in `.exe`, `.dll`, `.so`, `.dylib`, or native helper artifact exists.
