# v0.6.53 Go inspectPane Worker Lifecycle Slice

Result: v0.6.53 activates the first narrow worker lifecycle runtime slice: Go-owned read-only `inspectPane` only.

This slice follows the v0.6.52 worker lifecycle contract gate. It does not migrate broad worker lifecycle ownership.

## Capability Decision

Capability advertisement decision: `advertise-workerLifecycle-for-inspectPane-only`.

`workerLifecycle` is now an active helper capability only for `operation:"inspectPane"`.

This is advertised because the TypeScript kernel adapter preflights helper capabilities before calling helper methods. Advertising the capability keeps TypeScript preflight, Go health, packaged manifest, provenance, and resolver behavior consistent.

Unsupported worker lifecycle operations fail closed with compact diagnostics.

## Runtime Boundary

Go worker lifecycle authority is reachable only through `createAgentTeamKernelAdapter().inspectWorkerPane(...)`.

The TypeScript/pi facade remains authoritative. Tools/commands/hooks/renderers/TUI shell remain TS-owned. Leader-gated task governance, worker spawn policy, role separation, and diagnostics shaping remain TS-owned.

The new runtime path is intentionally narrow:

```text
TypeScript governance/facade -> core/kernel inspectWorkerPane(paneId) -> Go workerLifecycle operation:"inspectPane" -> compact pane inspection result
```

The result includes only compact inspection fields:

- `ok`
- `operation:"inspectPane"`
- `capability:"workerLifecycle"`
- `paneId` and `requestedPaneId`
- `exists`
- `currentCommand`
- `inMode`
- `mode`
- `copyMode`
- compact failure fields when unavailable
- `readOnly:true`
- `stateFilesRead:false`
- `stateFilesWritten:false`
- `tmuxMutation:false`

Diagnostics must not leak raw tmux stdout/stderr, cwd, stack traces, state archives, mailbox bodies, report bodies, raw terminal transcripts, or worker transcripts.

## Read-Only Proof

The Go helper uses read-only `tmux list-panes -a -F workerLifecycleInspectPaneFormat`.

No `display-message`, `send-keys`, `split-window`, `new-window`, or `kill-pane` is added to Go.

The only Go tmux commands after this slice are:

- existing v0.6.50 capture: `tmux list-panes -a -F tmuxPaneSnapshotFormat`.
- new v0.6.53 inspection: `tmux list-panes -a -F workerLifecycleInspectPaneFormat`.

No `createTeammatePane`, `wakePane`, `syncPaneLabels`, `killPane`, spawn, send-keys, label mutation, or pane mutation is migrated.

Unsupported operations fail closed, including:

- `listAgentTeamPanes`
- `wakePane`
- `syncPaneLabels`
- `createTeammatePane`
- `killPane`

## Preserved Boundaries

This slice does not migrate:

- worker spawn/create.
- wake delivery or terminal input.
- pane label mutation.
- pane kill/destructive lifecycle.
- state repository.
- task/report/PlanRun governance.
- team panel view-model.
- release/package verification.
- native artifact path or binary naming.

No worker-spawns-worker. No hidden scheduler/autopilot/background orchestration. No peer report auto-task creation.

The helper connection model remains per-call. Long-lived or pooled helper mode remains deferred behind the v0.6.52 prerequisites.

## Package And Release Guards

No package/release action is authorized in this slice:

- no `npm version`.
- no `npm publish`.
- no tag or GitHub release.
- no GitHub release assets.
- no package lockfiles.
- no `go.mod` or `go.sum`.
- no lifecycle hooks, postinstall downloads, or runtime `go build`.
- no native path rename.
- no binary rename.

`package.json` remains `0.6.8`.

The embedded native helper was rebuilt in-place only to keep helper binary, manifest, checksums, provenance, and attestation consistent with the newly advertised capability. The native path and binary name remain unchanged.
