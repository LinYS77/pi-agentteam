## Hosted workflow-dispatch observation runbook

This runbook is for a reviewer or team-lead observing a GitHub-hosted `workflow_dispatch` run after the exact commit has been pushed. Do not trigger or execute the hosted workflow from local implementation work, and do not check in hosted outputs.

1. Open `.github/workflows/go-helper-review-artifact.yml` in GitHub Actions and start or inspect a `workflow_dispatch` run for the exact pushed commit under review.
2. Record only the commit SHA and GitHub run id in the leader report or checkpoint note if the run is observed; do not commit downloaded bundles, verifier JSON, workflow summaries, generated manifests, checksums, provenance, attestations, or artifacts.
3. Confirm both jobs are green: `build-review-artifact` and `verify-review-artifact`.
4. Confirm there is exactly one required matrix row in both build and verify jobs: `ubuntu-latest / linux-x64-glibc`.
5. Confirm the verify job invokes `node scripts/verify-go-helper-artifact.cjs --artifact-root "$artifact_root"` with strict expected-context flags:
   - `--expected-target "${{ matrix.target }}"`
   - `--expected-source-revision "$expected_source_revision"`, where `expected_source_revision=$(git rev-parse --verify HEAD)` comes from the checked-out commit
   - `--expected-github-sha "${{ github.sha }}"`
   - `--expected-github-run-id "${{ github.run_id }}"`
   - `--json`
6. Confirm `sourceRevision` is compared to the checked-out `HEAD` and is not conflated with `github.sha`; pull request runs may use a merge ref.
7. Confirm artifact upload retention remains `retention-days: 7`.
8. Treat workflow output or any future `$GITHUB_STEP_SUMMARY` as review-only/non-availability evidence. If a summary is added later, it may only contain module, target, helper/protocol version, source revision, github sha/run id, review-only flags, and an explicit non-availability statement.

## Observation evidence handling

Allowed checkpoint evidence is limited to statements such as: exact commit SHA, GitHub run id, build job green, verify job green, one `ubuntu-latest / linux-x64-glibc` row, strict expected-context flags present, and artifact retention `7` days.

Do not commit or attach to a release any hosted run artifact, downloaded bundle, verifier JSON, workflow summary, `artifact-index.json`, `manifest.json`, `SHA256SUMS`, `provenance.json`, `LICENSE`, `license.json`, `attestation.intoto.jsonl`, native helper binary, generated checksum, generated provenance, generated attestation, tarball, zip, or package artifact.

Hosted observation is review-only evidence for the strict verifier/checkpoint. It is not release evidence, not install-source evidence, not package artifact evidence, not package-manager clean-install proof, and not normal-user native availability proof.
