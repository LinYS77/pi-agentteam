# v0.4.21 Go CI Package Artifact Prototype

> Scope: Slice 6 docs/tests prototype only. This prototypes future Go helper artifact, manifest, package dry-run, and clean-install smoke shapes using temporary test fixtures. It does not change `package.json`, add package metadata, add `optionalDependencies`, add lifecycle hooks, add package scripts, add lockfiles, add `go.mod`/`go.sum`, add checked-in native binaries, include `kernel/` in package contents, run `npm version`, run `npm publish`, commit, tag, push, make Go default, or delete the TypeScript parser fallback.

## Prototype Boundaries

Slice 6 is intentionally not a packaging implementation.

Allowed:

- create fake helper executable files under test temp directories.
- compute size and `sha256` during tests.
- write temporary manifest JSON and temporary package-layout files during tests.
- simulate `npm pack --dry-run --ignore-scripts` contents with test helper code.
- simulate clean install layout under temp directories.
- call the existing `go-packaged-preview` adapter with explicit `packagedHelperPath` injection.
- remove temp directories at test completion.

Not allowed:

- checked-in `.exe`, `.dll`, `.so`, `.dylib`, helper binaries, or generated package artifacts.
- changes to `package.json`, `package.json#files`, `optionalDependencies`, lifecycle hooks, or scripts.
- package lockfiles, root/helper `go.mod`, or root/helper `go.sum`.
- install-time Go builds, lifecycle downloads, implicit network fetches, or postinstall/preinstall/prepare behavior.
- `npm version`, `npm publish`, commit, tag, or push.
- default Go enablement, `go-cutover` expansion, or TypeScript parser fallback deletion.

## Generated Artifact Manifest Shape

The prototype manifest shape is generated in a test temp directory and must contain:

- `schemaVersion: 1`.
- `package.name` for a future helper artifact package.
- `package.version` matching the main package version, currently `0.6.8`.
- `helper.version` matching `0.3.0-read-model-shadow` or an approved successor.
- `helper.protocolVersion: 1`.
- `helper.module: "tmuxSnapshotParse"`.
- `helper.os`, `helper.arch`, and `helper.libc` platform tuple.
- `helper.filename` for the installed helper executable path inside the artifact package.
- `helper.size` from the generated helper file.
- `helper.sha256` from the generated helper file.
- `helper.executable: true` after executable-bit validation.
- `provenance.sourceRevision` as a placeholder such as `SOURCE_REVISION_PLACEHOLDER`.
- `provenance.generatedBy` as a CI/build placeholder.
- `provenance.attestation` as a provenance placeholder.
- `licenses` entries with package/helper license metadata, including `MIT` and a license file path.

The manifest is future-facing evidence only. It is not included in the real package by Slice 6.

## Temporary Artifact Creation

The test prototype creates a fake executable helper under a temp root such as `/tmp/agentteam-artifact-prototype-*`.

Required evidence:

- helper file exists only inside the temp root.
- executable bit is set and verified.
- size and `sha256` are computed from the temp file.
- manifest references the helper by package-relative filename.
- manifest validation rejects checksum mismatch, missing helper, non-executable helper, and unsupported platform tuples.
- temp root is removed after the suite.
- repository scan confirms no native/helper artifact remains under the source checkout.

## Package Dry-Run Shape

Slice 6 simulates a future generated artifact package contents list; it does not modify the real package and does not require `npm pack`.

Expected simulated package files:

- `package.json` for the temporary future artifact package fixture.
- `README.md` for package usage notes.
- `LICENSE` for license metadata.
- `manifest/agentteam-go-helper-manifest.json`.
- `bin/agentteam-tmux-snapshot-helper` or platform-equivalent helper filename inside the temp package layout.

Package dry-run guardrails:

- no `kernel/` source path appears in the simulated package contents.
- no lifecycle hooks appear in the temporary package manifest.
- no helper build/install/download/package/version/publish scripts appear.
- no `optionalDependencies` appear.
- only expected temp files are included.
- package metadata changes remain future work for a separate owner slice.

## Clean Install Smoke Shape

The clean-install smoke is simulated from a temp installed layout.

Required behavior:

- resolver lookup reads the temp manifest and validates platform tuple, size, `sha256`, and executable bit.
- helper path is returned only for supported, integrity-valid temp artifacts.
- the existing adapter uses the helper only with explicit `go-packaged-preview` plus packaged path injection.
- default/unset, disabled, typescript, go, auto, and current `go-cutover` behavior remain unchanged.
- direct `health` and `tmuxSnapshotParse` smoke succeeds for the temp helper artifact.
- no Go toolchain or manual source build is required by the simulated installed layout beyond creating the test fixture.
- `compactReadModelFingerprint` remains TypeScript fallback / non-cutover.

## Failure Scenarios

Slice 6 prototype tests cover:

- checksum mismatch maps to integrity failure and fails closed through current preview status mapping.
- unsupported platform shape maps to unsupported platform and fails closed through current preview status mapping.
- wrong helper version fails closed.
- wrong JSON-RPC protocol fails closed.
- missing `tmuxSnapshotParse` capability fails closed.
- non-executable helper fails manifest validation.
- preview packaged path does not silently invoke the TypeScript parser fallback for `tmuxSnapshotParse` failures.
- compact diagnostics do not leak helper path, stdout/stderr, repository/cwd paths, mailbox/report text, raw state/team JSON, sidecar/cache/index, worker prompts, stack traces, or package internals.

## STOP Conditions

STOP before any future package/native implementation if any of these are true:

- Slice 6 temp fixture behavior is treated as real package metadata or normal-user availability proof.
- real `package.json` metadata changes without a separate owner package slice.
- `optionalDependencies`, lifecycle hooks, helper scripts, lockfiles, Go modules, package file entries, or checked-in native/helper artifacts appear.
- generated artifacts are committed to the source repo.
- package dry-run includes raw `kernel/` source paths without a later approved package policy change.
- clean-install smoke requires a user Go toolchain, source checkout, lifecycle download, or install-time build.
- `go-packaged-preview` becomes default or activates outside explicit preview mode.
- current `go-cutover` starts using packaged discovery without explicit follow-up approval.
- `compactReadModelFingerprint` becomes cutover-owned.
- preview failures can silently invoke the TypeScript parser fallback.
- diagnostics leak forbidden fields.
- Go helper authority broadens beyond parser-only stdin/stdout `tmuxSnapshotParse`.
- `npm version`, `npm publish`, commit, tag, push, or Slice 7 starts without explicit instruction.
