# v0.6.80 Go Create Teammate Pane Cutover

Result: v0.6.80 cuts over only `tmux/panes.ts createTeammatePane(...)` pane discovery/creation/layout/resize behavior to Go-backed `workerLifecycle.createTeammatePane` behind the TypeScript facade.

This implements the runtime migration authorized by the v0.6.79 gate. It is a narrow mutating tmux cutover for teammate pane discovery, creation, layout, and leader resize only.

## Runtime Change

`ensureSwarmWindow(input.preferred, signal)` remains TypeScript-owned.

After `ensureSwarmWindow(...)` resolves the swarm target and leader pane id, `tmux/panes.ts createTeammatePane(...)` delegates pane discovery/creation/layout/resize to `createAgentTeamKernelAdapter().createTeammatePaneAsync(...)`.

`tmux/panes.ts createTeammatePane(...)` no longer calls direct TypeScript `runTmuxAsync(...)` for `list-panes`, `split-window`, `select-layout`, or `resize-pane`.

On compact helper failure, the TypeScript facade throws a compact `Error` from the validated helper failure reason, preserving the prior throwing create/layout failure shape without exposing raw tmux/helper output.

On success, the public return remains:

```ts
{ paneId, target }
```

Post-create label setting reuses the existing Go-backed `setPaneLabel(created.paneId, input.name, signal)` helper. This does not add a new label command surface. The existing v0.6.76 opaque Unicode label argv/no raw-label leakage policy remains unchanged.

Post-create label refresh reuses the existing Go-backed `refreshWindowPaneLabels(created.target, signal)` helper.

## Authorized Go Command Surface

Go `workerLifecycle.createTeammatePane` may execute only these pane discovery/creation/layout/resize tmux commands:

- `tmux list-panes -t <target> -F '#{pane_id}'`
- `tmux split-window -t <leaderPaneId> -h -p 34 [-c <cwd>] -P -F '#{pane_id}' [startCommand]`
- `tmux split-window -t <lastPaneId> -v [-c <cwd>] -P -F '#{pane_id}' [startCommand]`
- `tmux select-layout -t <target> main-vertical`
- `tmux select-layout -t <target> tiled`
- `tmux resize-pane -t <leaderPaneId> -x 66%`

No other Go tmux lifecycle commands are introduced by this slice.

The Go implementation validates the window target compactly before `list-panes` and `select-layout`. It validates pane ids compactly as `%123`-style ids before targeting leader/last pane ids and before returning a created pane id.

`cwd` and `startCommand` remain opaque high-risk argv-only values. They are passed as argv values only and are never shell-interpolated.

## Compact Diagnostics and No-Leak Policy

The adapter and helper use compact failure kinds such as:

- `invalid-target`
- `invalid-pane-id`
- `invalid-cwd`
- `invalid-start-command`
- `pane-not-found`
- `tmux-command-failed`
- `tmux-command-timeout`
- `tmux-unavailable`

Raw `cwd`, raw `startCommand`, raw tmux stdout/stderr, and raw helper output must not leak through diagnostics, errors, task reports, logs, or tests.

In compact policy terms: raw `cwd`, raw `startCommand`, raw tmux stdout/stderr, and raw helper output must not leak.

The TypeScript adapter ignores helper-provided `reason`/`error` text for `createTeammatePane` result validation and synthesizes compact diagnostics from the failure kind only, so a buggy helper cannot echo raw helper/stdout/stderr or raw opaque input text through adapter diagnostics.

## Preserved Existing Behavior and Boundaries

Still TypeScript-owned or out of scope:

- `ensureSwarmWindow(...)`
- `new-session`
- `new-window`

`new-session` and `new-window` remain TypeScript-owned in `tmux/windows.ts`.
- wake/send-keys
- kill-pane / kill-window / destructive lifecycle
- `clearPaneLabelsForTeam(...)` orchestration
- `syncPaneLabelsForTeam(...)` orchestration
- state repository
- task/report/PlanRun/mailbox/governance
- team panel/UI view-model
- release/package verification or package-manager native delivery

Already implemented operations remain unchanged:

- `setPaneLabel(paneId, label, signal)` remains v0.6.76 Go-backed with only pane-level `set-option -p` and `select-pane -T <label>` commands.
- `clearPaneLabel(paneId, signal)` remains v0.6.78 Go-backed with only pane-level `set-option -up` and `select-pane -T ''` commands.
- `markWindowAsAgentTeam(target, signal)` remains v0.6.72 Go-backed with only window marking `set-option -w` commands.
- `refreshWindowPaneLabels(target, signal)` remains v0.6.74 Go-backed with only pane-border window `set-option -w` commands.

`killPane(paneId)` and `clearPaneLabelSync(paneId)` remain TypeScript-owned synchronous helpers in `tmux/panes.ts`.
