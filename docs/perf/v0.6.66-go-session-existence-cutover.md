# v0.6.66 Go Session Existence Cutover

Result: v0.6.66 cuts over the `tmux/windows.ts` `ensureSwarmWindow()` session existence check from direct TypeScript `has-session` probing to a narrow Go-backed `workerLifecycle.sessionExists` operation through the cancellable async kernel adapter seam.

This slice removes only the read-only session-existence check. It does not move session/window creation, post-creation lookup, pane setup, marking, labels, or mutating lifecycle behavior into Go:

```text
tmux/windows.ts ensureSwarmWindow()
  -> createAgentTeamKernelAdapter().sessionExistsAsync(SWARM_SESSION, signal)
  -> workerLifecycle.sessionExists
  -> true only when session existence is positively confirmed
```

## Runtime Behavior

- `tmux/windows.ts` `ensureSwarmWindow()` now calls `createAgentTeamKernelAdapter().sessionExistsAsync(SWARM_SESSION, signal)`.
- The TypeScript `runTmuxNoThrowAsync(['has-session', '-t', SWARM_SESSION], undefined, signal).ok` implementation is removed from `ensureSwarmWindow()`.
- Go uses exactly `tmux has-session -t <sessionName>` for `workerLifecycle.sessionExists`.
- A positive helper result skips the existing TypeScript `new-session` branch as before.
- Missing session, helper failure, invalid response, empty session name, pre-aborted signal, and in-flight abort fail closed to `false`, so the existing TypeScript creation path remains in charge.
- Public behavior of `ensureSwarmWindow(preferred?, signal?)` is preserved from the caller perspective.
- Raw stdout, raw stderr, cwd, stack, helper paths, mailbox/report bodies, worker transcript bodies, and full paths are not exposed.

## Preserved Boundaries

- `new-session`, `new-window`, post-creation `list-windows -F '#{window_id}\t#{window_name}'`, pane setup `list-panes`, marking, labels, kill, state/task/UI/release/package remain TypeScript-owned; inside-tmux current binding `display-message` fallbacks are superseded by the v0.6.67 `captureCurrentPaneBinding()` reuse cutover, and detached target-based leader-pane `display-message` is superseded by the v0.6.68 `resolvePaneBindingAsync()` reuse cutover.
- `markWindowAsAgentTeam()` and `refreshWindowPaneLabels()` remain TypeScript-owned.
- `createTeammatePane()`, pane splitting, pane labels, `wakePane()`, and `killPane()` remain TypeScript-owned.
- `findAgentTeamWindowTarget()` remains the v0.6.65 Go-backed window discovery facade.
- `waitForPaneAppStart()` remains the v0.6.64 Go-backed inspect polling facade.
- `ensureTmuxAvailable()` remains the v0.6.63 Go-backed tmux availability facade.
- The native helper path and binary name remain unchanged.

Because Go source changes, the existing embedded helper is rebuilt in the same approved path with refreshed manifest, checksums, provenance, and placeholder attestation.

`package.json` remains `0.6.8`.
