# v0.6.54 Go listAgentTeamPanes Worker Lifecycle Slice

Result: v0.6.54 activates the next narrow worker lifecycle runtime slice: Go-owned read-only `listAgentTeamPanes` alongside the existing read-only `inspectPane`.

This slice follows v0.6.53. It does not migrate broad worker lifecycle ownership.

## Capability Decision

Capability advertisement decision: `advertise-workerLifecycle-for-read-only-inspect-and-list-agentteam-panes`.

`workerLifecycle` remains one active helper capability for exactly two read-only operations:

- `operation:"inspectPane"`
- `operation:"listAgentTeamPanes"`

Unsupported worker lifecycle operations continue to fail closed with compact diagnostics.

## Runtime Boundary

Go worker lifecycle authority is reachable only through explicit TypeScript kernel adapter seams:

- `createAgentTeamKernelAdapter().inspectWorkerPane(...)`
- `createAgentTeamKernelAdapter().listAgentTeamPanes()`

The TypeScript/pi facade remains authoritative. Tools/commands/hooks/renderers/TUI shell remain TS-owned. Leader-gated task governance, worker spawn policy, role separation, and diagnostics shaping remain TS-owned.

The new runtime path is intentionally narrow:

```text
TypeScript governance/facade -> core/kernel listAgentTeamPanes() -> Go workerLifecycle operation:"listAgentTeamPanes" -> compact pane list result
```

The list result includes only panes with a non-empty `paneId` and non-empty `label` (`@agentteam-name`), matching the TypeScript `tmux/core.ts` `listAgentTeamPanes()` semantics. `byPaneId` is built from the filtered list only, so unlabeled/non-agentteam panes are excluded from the worker lifecycle seam.

The list result includes only compact pane identity fields:

- `ok`
- `operation:"listAgentTeamPanes"`
- `capability:"workerLifecycle"`
- `panes`
- `byPaneId`
- compact pane fields: `paneId`, `target`, `label`, `currentCommand`
- compact failure fields when unavailable
- `readOnly:true`
- `stateFilesRead:false`
- `stateFilesWritten:false`
- `tmuxMutation:false`

Diagnostics must not leak raw tmux stdout/stderr, cwd, stack traces, state archives, mailbox bodies, report bodies, raw terminal transcripts, or worker transcripts.

## Read-Only Proof

The Go helper uses only read-only `tmux list-panes -a -F ...` commands.

For `listAgentTeamPanes`, Go reuses the existing compact snapshot format: `tmux list-panes -a -F tmuxPaneSnapshotFormat`, then filters out rows without a pane id or `@agentteam-name` label.

For `inspectPane`, Go keeps the existing v0.6.53 format: `tmux list-panes -a -F workerLifecycleInspectPaneFormat`.

No `display-message`, `send-keys`, `split-window`, `new-window`, `kill-pane`, `set-option`, `select-pane`, or `respawn-pane` is added to Go.

No `createTeammatePane`, `wakePane`, `syncPaneLabels`, `killPane`, spawn, send-keys, label mutation, or pane mutation is migrated.

Unsupported operations fail closed, including:

- `wakePane`
- `syncPaneLabels`
- `createTeammatePane`
- `killPane`

## Preserved Boundaries

This slice does not migrate:

- worker spawn/create.
- wake delivery or terminal input.
- pane label mutation.
- pane kill/destructive lifecycle.
- state repository.
- task/report/PlanRun governance.
- team panel view-model.
- release/package verification.
- native artifact path or binary naming.

No worker-spawns-worker. No hidden scheduler/autopilot/background orchestration. No peer report auto-task creation.

The helper connection model remains per-call. Long-lived or pooled helper mode remains deferred behind the v0.6.52 prerequisites.
