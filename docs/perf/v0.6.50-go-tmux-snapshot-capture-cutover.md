# v0.6.50 Go tmuxSnapshotCapture Cutover

Result: v0.6.50 implements the first post-v0.6.49 control-plane expansion slice: Go now owns the narrow tmux snapshot capture adapter.

This slice is intentionally limited to `tmuxSnapshotCapture`:

```text
TypeScript caller -> Go `tmux list-panes -a -F <format>` -> Go snapshot parse -> compact TmuxSnapshot result
```

## Runtime Behavior

- `tmux/snapshot.ts` keeps `TMUX_PANE_SNAPSHOT_FORMAT` as the TypeScript-visible protocol constant.
- `captureTmuxSnapshot(capturedAt)` now delegates to `createAgentTeamKernelAdapter().captureTmuxSnapshot(capturedAt)`.
- The embedded Go helper exposes `tmuxSnapshotCapture` in health capabilities.
- The Go helper executes exactly `tmux list-panes -a -F tmuxPaneSnapshotFormat` for this capability.
- The Go helper reuses the Go `tmuxSnapshotParse` parser for successful capture output.
- Missing helper, incompatible helper, tmux unavailable, tmux command failure, and timeout return compact unknown/stale diagnostics.

## Preserved Boundaries

This slice does not move these responsibilities into Go:

- worker pane creation, wake delivery, labeling, or kill lifecycle.
- `display-message`, `send-keys`, `split-window`, `new-window`, or broad tmux lifecycle commands.
- state/repository reads or writes.
- task/report/PlanRun governance.
- mailbox/report full-text reads.
- `/team` TUI rendering.
- package versioning, npm publishing, tags, GitHub releases, signing, or second-platform support.

TypeScript/pi remains the public facade for extension registration, tool/command schemas, prompts, hooks, and TUI integration.

## Failure Policy

`tmuxSnapshotCapture` failure is explicit parser/capture-unavailable state, not a successful empty pane list.

Failure snapshots must include:

- `ok:false`
- `status:"unknown"`
- `resultMarker:"stale"`
- `module:"tmuxSnapshotCapture"`
- `capability:"tmuxSnapshotCapture"`
- a compact `cutoverFailureKind`

Diagnostics must not leak raw stdout, stderr, cwd, stack traces, state archives, mailbox bodies, report bodies, or worker transcripts.

## Package And Release Guards

No package/release action is authorized in this slice:

- no `npm version`.
- no `npm publish`.
- no tag or GitHub release.
- no package lockfiles.
- no `go.mod` or `go.sum`.
- no lifecycle hooks, postinstall downloads, or runtime `go build`.
- no second-platform support or signing release gate.

`package.json` remains `0.6.8`.

The v0.6.50 guard suite verifies:

- `captureTmuxSnapshot` delegates through the kernel adapter.
- Go exposes and handles `tmuxSnapshotCapture`.
- Go tmux execution is limited to `list-panes` snapshot capture.
- capture failures are unknown/stale and no-leak.
- worker lifecycle, state, task/report/PlanRun, UI, and release/package authority remain unmigrated.
- package version and release surfaces remain unchanged.
