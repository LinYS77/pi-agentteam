# v0.6.84 Go Detached New-Window Cutover

Result: v0.6.84 cuts over only the detached missing-agentteam-window `tmux/windows.ts ensureSwarmWindow(...)` `new-window` command to Go-backed `workerLifecycle.createDetachedSwarmWindow` behind the TypeScript facade.

This implements the runtime migration authorized by the v0.6.83 gate. It is a narrow mutating tmux cutover for detached swarm window creation only.

## Runtime Change

`ensureSwarmWindow(preferred, signal)` remains TypeScript-owned.

Detached missing-session creation remains the v0.6.82 Go-backed cutover and is unchanged:

- `createAgentTeamKernelAdapter().createDetachedSwarmSessionAsync(SWARM_SESSION, SWARM_WINDOW, signal)`

After detached session handling, the current agentteam window discovery still decides whether an agentteam window already exists:

```ts
let initialTarget = await findAgentTeamWindowTarget(SWARM_SESSION, signal)
```

When `initialTarget` is absent, `tmux/windows.ts ensureSwarmWindow(...)` now delegates only the missing-window creation step to:

- `createAgentTeamKernelAdapter().createDetachedSwarmWindowAsync(SWARM_SESSION, SWARM_WINDOW, signal)`

`tmux/windows.ts ensureSwarmWindow(...)` no longer calls direct TypeScript `runTmuxAsync(...)` for the detached missing-window `new-window` command.

On compact helper failure, the TypeScript facade throws a compact `Error` from the validated helper failure reason, preserving the prior throwing create failure behavior without exposing raw tmux/helper output.

Post-creation lookup remains unchanged and TypeScript-orchestrated:

- `findWindowTargetByName(SWARM_SESSION, SWARM_WINDOW, signal)`
- `throw new Error('Failed to locate agentteam tmux window after creation')`
- `markWindowAsAgentTeam(initialTarget, signal)`

Final pane/binding lookup remains unchanged and TypeScript-orchestrated:

- `firstPaneInWindow(initialTarget, signal)`
- `resolvePaneBindingAsync(leaderPaneId, signal)`
- final `markWindowAsAgentTeam(target, signal)`
- `refreshWindowPaneLabels(target, signal)`

The inside-tmux branch remains TypeScript-owned and unchanged.

## Authorized Go Command Surface

Go `workerLifecycle.createDetachedSwarmWindow` may execute only this tmux command:

- `tmux new-window -t <SWARM_SESSION> -n <SWARM_WINDOW>`

No other Go tmux lifecycle commands are introduced by this slice.

Future/current Go passes the session and window names as argv values only, never shell-interpolates them.

The Go implementation compactly validates the session name and window name before executing tmux.

## Compact Diagnostics and No-Leak Policy

The adapter and helper use compact failure kinds such as:

- `invalid-session`
- `invalid-window-name`
- `tmux-command-failed`
- `tmux-command-timeout`
- `tmux-unavailable`

Raw tmux stdout/stderr/helper output must not leak through diagnostics, errors, task reports, logs, or tests.

In compact policy terms: raw tmux stdout/stderr/helper output must not leak.

The TypeScript adapter ignores helper-provided `reason`/`error` text for `createDetachedSwarmWindow` result validation and synthesizes compact diagnostics from the failure kind only, so a buggy helper cannot echo raw helper/stdout/stderr through adapter diagnostics.

## Preserved Existing Behavior and Boundaries

Still TypeScript-owned or out of scope:

- `ensureSwarmWindow(...)` broader orchestration
- detached `new-session` behavior beyond the unchanged v0.6.82 Go-backed call
- inside-tmux branch behavior
- post-creation window lookup
- `markWindowAsAgentTeam(...)` behavior beyond the unchanged post-create call
- `refreshWindowPaneLabels(...)`
- `createTeammatePane(...)`, pane split/layout/resize
- wake/send-keys
- kill-pane / kill-window / destructive lifecycle
- state repository
- task/report/PlanRun/mailbox/governance
- team panel/UI view-model
- release/package control plane

Already implemented operations remain unchanged:

- `createDetachedSwarmSessionAsync(SWARM_SESSION, SWARM_WINDOW, signal)` remains the v0.6.82 Go-backed detached missing-session `new-session` cutover and is not changed here.
- `createTeammatePane(...)` remains the v0.6.80 Go-backed pane split/layout/resize cutover and is not changed here.
- `setPaneLabel(paneId, label, signal)` remains v0.6.76 Go-backed with only pane-level `set-option -p` and `select-pane -T <label>` commands.
- `clearPaneLabel(paneId, signal)` remains v0.6.78 Go-backed with only pane-level `set-option -up` and `select-pane -T ''` commands.
- `markWindowAsAgentTeam(target, signal)` remains v0.6.72 Go-backed with only window marking `set-option -w` commands.
- `refreshWindowPaneLabels(target, signal)` remains v0.6.74 Go-backed with only pane-border window `set-option -w` commands.

After this cutover, `tmux/windows.ts ensureSwarmWindow(...)` does not retain a hidden TypeScript fallback for the same detached missing-window `new-window` behavior.
