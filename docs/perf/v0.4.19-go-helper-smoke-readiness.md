# v0.4.19 Go Helper Smoke Readiness

> Scope: Slice 4 docs/reference commands only. These commands are optional local source-only reviewer smokes; they do not change runtime behavior, delete TypeScript fallback paths, make Go default or required, change `package.json` version, add package scripts, run `npm version`, run `npm publish`, add lifecycle hooks, add package lock files, add `go.mod`/`go.sum`, add checked-in native binaries, include `kernel/` in the package, or add native packaging.

## Purpose

v0.4.19 keeps Go helper readiness source-only. This document normalizes reviewer commands for local helper smoke checks so reviewers use the same temporary build path, environment variables, cleanup, expected signals, and package/native sanity checks without turning the helper into a packaged/default runtime.

These smokes are Model A / Model B evidence for `docs/perf/v0.4.19-go-runtime-prerequisites.md` and are summarized by the final readiness checkpoint in `docs/perf/v0.4.19-go-kernel-readiness-checkpoint.md`:

- Model A: source-only/manual helper path for GitHub-only readiness.
- Model B: explicit user-provided helper path for experimental/local cutover smoke only.
- Not Model C: no native packaging matrix, no shipped/default Go runtime, and no fallback deletion approval.

## Source-Only Rules

- Build helper binaries only into `/tmp` using `mktemp /tmp/agentteam-v0419-kernel.XXXXXX` or an equivalent temporary path.
- Build with `GO111MODULE=off` from `kernel/go/agentteam-kernel`.
- Use the helper only through explicit `PI_AGENTTEAM_KERNEL=go` and `PI_AGENTTEAM_KERNEL_HELPER="$helper"` for the one smoke command being reviewed.
- Remove the temporary helper with `rm -f "$helper"` after the smoke.
- Do not check in the binary, copy it into the repo, add a package script, add an npm lifecycle hook, add a postinstall/download step, or add native packaging; shorthand: no postinstall/download.
- Missing Go toolchain means optional-skip/manual-smoke unavailable; it is not a failure of the default TypeScript runtime.

## Temporary Helper Build

Run from the repository root:

```bash
helper="$(mktemp /tmp/agentteam-v0419-kernel.XXXXXX)"
(cd kernel/go/agentteam-kernel && GO111MODULE=off go build -o "$helper" .)
```

Expected build posture:

- `$helper` is under `/tmp`.
- no file is written under `kernel/` except normal source files already in the repository.
- no `go.mod`, `go.sum`, package lock, native artifact, package script, npm lifecycle hook, or package metadata change is introduced.

Always clean up:

```bash
rm -f "$helper"
```

A safe one-shot shell pattern is:

```bash
helper="$(mktemp /tmp/agentteam-v0419-kernel.XXXXXX)"
trap 'rm -f "$helper"' EXIT
(cd kernel/go/agentteam-kernel && GO111MODULE=off go build -o "$helper" .)
PI_AGENTTEAM_KERNEL=go PI_AGENTTEAM_KERNEL_HELPER="$helper" AGENTTEAM_BENCH_ITERATIONS=1 npm run --silent bench:state-read-model \
  > /tmp/agentteam-v0419-state-go-helper.json
rm -f "$helper"
trap - EXIT
```

## Optional Direct JSON-RPC Smokes

Use direct helper smokes only when the local Go toolchain is available. These commands are source-only and do not use npm package hooks:

```bash
printf '{"jsonrpc":"2.0","id":"health-1","method":"health"}\n' \
  | (cd kernel/go/agentteam-kernel && GO111MODULE=off go run .)

printf '%s\n' '{"jsonrpc":"2.0","id":"snapshot-1","method":"tmuxSnapshotParse","params":{"stdout":"%1\tsession:@1\tagentteam worker\tpi\n","capturedAt":1700000000000}}' \
  | (cd kernel/go/agentteam-kernel && GO111MODULE=off go run .)

printf '%s\n' '{"jsonrpc":"2.0","id":"read-model-1","method":"compactReadModelFingerprint","params":{"input":{"mode":"attached","team":{"name":"demo","leaderCwd":"/tmp/demo"},"members":[],"tasks":[],"mailbox":[]}}}' \
  | (cd kernel/go/agentteam-kernel && GO111MODULE=off go run .)

(cd kernel/go/agentteam-kernel && GO111MODULE=off go test .)
```

Expected health signals:

- `protocolVersion` is `1`.
- `helperVersion` is compatible and currently `0.3.0-read-model-shadow` unless a future docs/test slice explicitly records a successor.
- `capabilities` include `health`, `profile`, `tmuxSnapshotParse`, and `compactReadModelFingerprint`.
- `businessPathsConnected` is `false`.

Expected direct smoke signals:

- `tmuxSnapshotParse` parses TypeScript-captured stdout only; it does not execute tmux.
- `compactReadModelFingerprint` returns `readOnly:true`, `fullTextIncluded:false`, `stateFilesRead:false`, and `stateFilesWritten:false`.
- Direct smokes do not create runtime `/team` diagnostics, state writes, repository writes, sidecar/outbox writes, package changes, or native artifacts.

## Optional Helper-Backed Bench Smoke

Run only when the local Go toolchain is available:

```bash
helper="$(mktemp /tmp/agentteam-v0419-kernel.XXXXXX)"
trap 'rm -f "$helper"' EXIT
(cd kernel/go/agentteam-kernel && GO111MODULE=off go build -o "$helper" .)
PI_AGENTTEAM_KERNEL=go PI_AGENTTEAM_KERNEL_HELPER="$helper" AGENTTEAM_BENCH_ITERATIONS=1 npm run --silent bench:state-read-model \
  > /tmp/agentteam-v0419-state-go-helper.json
rm -f "$helper"
trap - EXIT
```

Expected helper-backed bench signals in `/tmp/agentteam-v0419-state-go-helper.json`:

- `kernel.enabled` is `true` with the explicit helper.
- `kernel.mode` is `go`.
- `kernel.fallbacks` is `0` and no `fallbackKind` is present.
- parity is matched where applicable, including `parityMatched:true` for read-model shadow evidence.
- `readOnly:true`, `fullTextIncluded:false`, `stateFilesRead:false`, and `stateFilesWritten:false` remain true/false as documented.
- no runtime `/team` diagnostics or fallback diagnostics are introduced.
- the helper remains source-only/manual and is removed from `/tmp` after the run.

## Missing Go Toolchain

If `go version` is unavailable, record the helper smoke as optional-skip/manual-smoke unavailable:

```bash
if ! go version >/dev/null 2>&1; then
  echo 'optional Go helper smoke skipped: local Go toolchain unavailable'
fi
```

Expected skip posture:

- default TypeScript runtime is still valid.
- `PI_AGENTTEAM_KERNEL` remains unset or TypeScript/disabled by default.
- no reviewer should mark default TypeScript runtime failed solely because local Go is missing.
- no package/native workaround should be introduced to hide a missing local Go toolchain.

## Package/Native Sanity

Run from the repository root:

```bash
node - <<'NODE'
const fs = require('fs')
const pkg = JSON.parse(fs.readFileSync('package.json', 'utf8'))
const lifecycle = ['preinstall','install','postinstall','prepare','prepublish','prepublishOnly','publish','postpublish']
  .filter(name => pkg.scripts && Object.prototype.hasOwnProperty.call(pkg.scripts, name))
if (pkg.version !== '0.6.8') throw new Error(`unexpected package version ${pkg.version}`)
if ((pkg.files || []).some(entry => entry === 'kernel' || entry.startsWith('kernel/') || entry.includes('/kernel/'))) throw new Error('package files include kernel')
if (lifecycle.length) throw new Error(`lifecycle scripts present: ${lifecycle.join(', ')}`)
for (const file of ['package-lock.json', 'npm-shrinkwrap.json', 'go.mod', 'go.sum', 'kernel/go/agentteam-kernel/go.mod', 'kernel/go/agentteam-kernel/go.sum']) {
  if (fs.existsSync(file)) throw new Error(`${file} exists`)
}
console.log('package/native config sanity passed')
NODE

find . -type f \( -name '*.exe' -o -name '*.dll' -o -name '*.so' -o -name '*.dylib' \) \
  -not -path './node_modules/*' -not -path './.git/*'
```

Expected package/native signals:

- `package.json` version remains `0.6.8`.
- `package.json#files` excludes `kernel/`.
- no `preinstall`, `install`, `postinstall`, `prepare`, publish, or prepublish lifecycle hooks exist.
- no package script is added for helper build/download/smoke.
- no `package-lock.json`, `npm-shrinkwrap.json`, root or helper `go.mod`, root or helper `go.sum`, checked-in `.exe`, `.dll`, `.so`, `.dylib`, or other native artifact appears.

## Non-Goals

This Slice 4 command normalization does not approve or implement:

- package scripts for helper build, smoke, install, download, or postinstall behavior.
- runtime behavior changes, Go default/runtime-required behavior, runtime `/team` diagnostics, or fallback deletion.
- native packaging, checked-in binaries, package manager lifecycle hooks, npm version changes, npm publish, package lock files, `go.mod`, `go.sum`, commit, tag, push, or `kernel/` package inclusion.
- Go tmux execution, pane lifecycle, worker lifecycle, state writes, repository writes, sidecar/outbox writes, task/report governance, PlanRun authority, full-text boundary movement, or package/release control.
