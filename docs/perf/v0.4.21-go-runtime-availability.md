# v0.4.21 Go Runtime Availability Decision Matrix

> Scope: Slice 1 Model C0 decision/signoff planning only. This document does not implement a runtime resolver, native packaging, helper downloads, install-time builds, default Go runtime, npm release, package metadata changes, or TypeScript fallback deletion.

## Context

v0.4.20 completed the GitHub-only `go-cutover` checkpoint in `docs/perf/v0.4.20-go-cutover-checkpoint.md`. That checkpoint accepted experimental local evidence for the `tmuxSnapshotParse` Go-owned cutover after leader review, but it explicitly kept npm/default/native cutover stopped.

Current facts carried forward from v0.4.19 and v0.4.20:

- `PI_AGENTTEAM_KERNEL=go-cutover` remains explicit/local-only and must be paired with an explicit helper path for reviewer/local smoke.
- npm/default/native cutover remains STOP.
- `tmuxSnapshotParse` is the only cutover-owned module.
- `compactReadModelFingerprint` remains TypeScript fallback / non-cutover under `go-cutover`.
- Model B explicit helper path remains local/reviewer-only and cannot justify deleting the TypeScript parser fallback for default users.
- v0.4.19 runtime prerequisites in `docs/perf/v0.4.19-go-runtime-prerequisites.md` still block shipped/default fallback deletion until normal-user runtime availability is signed off.

## Decision

v0.4.21 Slice 1 is Model C0: decision/signoff docs/tests only.

Preferred future Model C direction:

- Native companion packages per supported platform, or generated release artifacts with checksums/provenance, are the preferred direction for normal-user helper availability.
- Main-package bundled prebuilt binaries are a possible second-choice design if package size, provenance, platform support, executable permissions, and release workflow are accepted.
- Postinstall/preinstall/prepare downloads and install-time `go build` are rejected for this signoff path unless a separate security/offline/provenance approval explicitly changes that policy.
- Source package / user-built helper remains useful for development, but it does not satisfy normal-user runtime availability.

Checkpoint recommendation: GO for Model C0 docs/tests signoff; STOP for native implementation, npm/default cutover, and TypeScript fallback deletion.

## Minimum Normal-User Evidence

Before any future packaged/default Go-owned parser can be accepted, reviewers must see normal-user availability evidence, not just local helper-path smokes.

Required evidence:

- Fresh install on every supported OS/arch can locate and execute a compatible helper without a Go toolchain, source checkout, manual `/tmp` build, or `PI_AGENTTEAM_KERNEL_HELPER` override.
- Platform matrix is explicit: OS, CPU architecture, Linux libc target when relevant, executable permission expectations, tmux support assumptions, pi extension assumptions, and unsupported-platform behavior.
- Unsupported-platform policy is explicit. If AgentTeam still supports a platform where no helper is available, global default cutover and global TypeScript fallback deletion remain STOP for that platform unless platform support is narrowed.
- Helper health/protocol/capability/version check passes from installed artifacts: protocol version `1`, helper version `0.3.0-read-model-shadow` or an approved successor, `tmuxSnapshotParse` capability, and `businessPathsConnected:false`.
- Package/helper version skew is detected and fails closed safely.
- Missing, corrupted, wrong-platform, blocked, non-executable, incompatible, or unsupported helper cases fail closed with compact diagnostics and no silent TypeScript parser fallback after a future default cutover.
- Offline/CI behavior is documented, including tarball/cache workflow for companion packages or generated artifacts and the expected behavior when optional native packages are unavailable.
- Install and uninstall/upgrade behavior is documented, including executable-bit validation and cleanup expectations.

## Package/Native Policy for Slice 1

v0.4.21 Slice 1 makes no package behavior changes.

Required unchanged state:

- `package.json` version remains `0.6.8`.
- no `npm version`.
- no `npm publish`.
- no npm lifecycle hooks: `preinstall`, `install`, `postinstall`, `prepare`, `prepublish`, `prepublishOnly`, `publish`, or `postpublish`.
- no helper build/download/package scripts.
- no `package-lock.json` or `npm-shrinkwrap.json`.
- no root or helper `go.mod`.
- no root or helper `go.sum`.
- no checked-in native binaries or helper artifacts.
- no `kernel/` package inclusion.
- no package metadata change for companion packages, optional dependencies, generated artifacts, resolver paths, or native binaries.

Future prebuilt helpers can be acceptable only after signoff and only as generated release/package artifacts with checksums, provenance, license metadata, executable-bit validation, version compatibility checks, and clean uninstall/upgrade behavior. They must not be committed into the source repo as native artifacts.

## Diagnostics and Fallback Deletion Gates

No TypeScript runtime fallback deletion is allowed until normal-user availability signoff passes.

Future shipped/default diagnostics may need a compact `/team` signal because parser unavailable under a default Go-owned path could otherwise look like stale pane data. That design is not implemented in Slice 1.

Allowed future diagnostic shape:

- module `tmuxSnapshotParse`,
- status/result marker `unknown`/`stale`,
- stable compact failure kind such as `missing-helper` or a future native-specific `unsupported-platform`,
- short remediation text,
- supported-platform hint,
- reinstall/rollback pointer.

Forbidden diagnostic leaks:

- raw helper path,
- helper stdout/stderr bodies,
- repository path or cwd path,
- mailbox/report text,
- raw `cutoverReason`,
- raw team JSON or state files,
- sidecar/cache/index contents,
- hidden runtime state,
- worker prompts,
- stack traces,
- package/release internals beyond a compact remediation pointer.

Rollback for default/native cutover must be via GitHub tag/npm corrected release, not a hidden runtime TypeScript parser fallback.

## STOP Conditions

STOP npm/default/native cutover if any of these are true:

- helper availability is only source-only, explicit-helper-path, manual `/tmp` build, local Go-toolchain based, or reviewer-only.
- any supported platform lacks a normal install helper path.
- unsupported-platform policy is unresolved.
- native packaging depends on postinstall/preinstall/prepare downloads or install-time `go build` without a separate security/offline/provenance approval.
- helper binaries lack checksum/provenance/version compatibility checks.
- package/helper version skew is not detected.
- missing, incompatible, corrupted, wrong-platform, blocked, or non-executable helper can silently invoke the TypeScript parser fallback in a default cutover path.
- parser failure can cause pane loss, worker error, kill panes, force reconcile, or false successful empty snapshots.
- diagnostics leak helper paths, stdout/stderr, repo paths, mailbox/report text, raw state, sidecar/cache/index, hidden runtime state, worker prompts, or raw `cutoverReason`.
- Go gains tmux execution/capture, state writes, worker lifecycle, PlanRun/governance, full-text boundary, or package/release control.
- `compactReadModelFingerprint` becomes cutover-owned before `tmuxSnapshotParse` normal-user availability is solved and separately approved.
- `package.json` version changes, npm publish/version is proposed, `kernel/` is packaged, lifecycle hooks are added, lockfiles/go modules/native artifacts appear, or helper build/download/package scripts are added during Slice 1.
- downstream runtime/package implementation starts before leader-approved signoff for later slices.
