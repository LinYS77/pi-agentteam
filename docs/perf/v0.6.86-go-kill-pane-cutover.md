# v0.6.86 Go Kill-Pane Cutover

Result: v0.6.86 cuts over only `tmux/panes.ts killPane(paneId)` from the direct TypeScript `runTmuxNoThrow(['kill-pane', '-t', paneId])` behavior to a Go-backed `workerLifecycle.killPane` operation behind the TypeScript facade.

## Runtime contract

- `killPane(paneId): void` remains synchronous, public no-throw, and returns no value.
- The TypeScript facade delegates to `createAgentTeamKernelAdapter().killPane(paneId)` inside a no-throw wrapper.
- The hidden direct TypeScript fallback for this behavior is removed: `killPane(...)` no longer calls `runTmuxNoThrow(['kill-pane', '-t', paneId])`.
- Adapter/helper/tmux/timeout/invalid-pane failures produce compact fail-closed helper results and are swallowed by the public facade to preserve existing semantics.
- `clearPaneLabelSync(paneId)` remains TypeScript-owned and unchanged.

## Authorized Go command surface

The new Go command surface is exactly one destructive tmux command shape:

- `tmux kill-pane -t <paneId>`

Constraints:

- The helper must invoke tmux with argv execution only, not shell interpolation.
- The pane id must be compactly validated as a `%123`-style tmux pane id before execution.
- Failure diagnostics must use compact synthesized `failureKind` / `reason` / `error` values.
- Raw pane input, raw tmux stdout/stderr, helper paths, stack traces, and raw helper output must not leak through adapter results or the public facade.

## Explicit non-goals

Out of scope for v0.6.86:

- `clearPaneLabelSync(paneId)` migration or deletion.
- `clearAndKillTeamPanes(...)`, `killTeamPanes(...)`, or team kill orchestration changes.
- `wakePane(...)`, `send-keys`, `kill-window`, `kill-session`, `respawn-pane`, buffers, or broader destructive tmux lifecycle.
- State repository, task/report/PlanRun/mailbox governance, team panel/UI, release/package controls, normal-user native delivery, or package publishing.
- Native artifact path/name changes.

## Artifact/package guardrails

- Existing native helper path/name stays `native/tmuxSnapshotParse/0.3.0-read-model-shadow/linux-x64-glibc/agentteam-tmuxSnapshotParse`.
- The helper artifact is rebuilt in place only to carry the new operation and refreshed metadata/checksums/provenance/attestation.
- `package.json` remains `0.6.8`.
- No lockfiles, `go.mod`, `go.sum`, lifecycle hooks, postinstall/download/native dependency flow, npm version, npm publish, release tags, or release assets are added.
