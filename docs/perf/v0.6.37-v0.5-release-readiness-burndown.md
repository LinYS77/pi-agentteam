## Slice 1 — P0 bug inventory & severity ledger

Slice 1 records P0 seams, current status, existing evidence, release impact, and the required proof or next action before a v0.5 release decision. It is a docs/tests guard only.

Status meaning:

- `green`: concrete existing docs/tests evidence exists and the row identifies required proof; this is not a release-ready claim.
- `watch`: partial evidence exists, but final proof or release threshold evidence is still required.
- `blocked`: the seam cannot advance without an explicit later decision or missing prerequisite outside this slice.

- Relative fallback rule: if the absolute threshold cannot be judged stably on the current reviewer machine, the candidate must show at least `>= 50% improvement` over a reviewer-accepted baseline artifact recorded with matching fixture, command, warmup/measured iterations, metric field, and environment metadata.
