# v0.6.64 Go Pane App-Start Wait Cutover

Result: v0.6.64 cuts over `tmux/process.ts` `waitForPaneAppStart(paneId, timeoutMs, signal)` from direct TypeScript target-based `display-message` polling to the existing Go-backed `workerLifecycle.inspectPane` async adapter path.

This slice removes the last direct pane-current-command polling call from `tmux/process.ts` without changing worker spawn semantics or adding Go/native surface:

```text
tmux/process.ts waitForPaneAppStart(paneId, timeoutMs, signal)
  -> createAgentTeamKernelAdapter()
  -> kernel.inspectWorkerPaneAsync(paneId, signal) on each poll
  -> boolean readiness result
```

## Runtime Behavior

- `tmux/process.ts` now creates a kernel adapter once per wait loop and calls `kernel.inspectWorkerPaneAsync(paneId, signal)` on each poll.
- The TypeScript `runTmuxNoThrowAsync(['display-message', '-p', '-t', paneId, '#{pane_current_command}'], undefined, signal)` fallback is removed from `waitForPaneAppStart()`.
- Polling cadence is preserved as a loop with sleeps capped at 200ms between inspect calls.
- `SHELL_COMMANDS` remains the filter: a trimmed non-empty command not in the shell set returns `true`.
- Timeout, shell command, empty command, missing pane, helper failure, empty pane id, pre-aborted signal, and in-flight abort all return `false` without throwing.
- Public output remains boolean-only; raw stdout, raw stderr, cwd, stack, helper paths, mailbox/report bodies, worker transcript bodies, and full paths are not exposed.

## Preserved Boundaries

- Go source and native helper artifacts do not change for v0.6.64 because the existing `workerLifecycle.inspectPane` contract already returns compact `currentCommand`.
- No target-based Go `display-message` is added.
- `workerLifecycle.inspectPane` remains the global read-only `tmux list-panes -a -F workerLifecycleInspectPaneFormat` path.
- Spawn, labels, kill, window/session creation, state repository, task/report/PlanRun, team panel view-model, and release/package verification remain TypeScript-owned.
- The native helper path and binary name remain unchanged.

`package.json` remains `0.6.8`.

## Guard Evidence

The v0.6.64 guard suite verifies:

- `waitForPaneAppStart()` delegates polling to `kernel.inspectWorkerPaneAsync(paneId, signal)`.
- the direct TypeScript target-based `display-message` pane-current-command fallback is removed.
- shell commands keep polling until timeout or until a non-shell command appears.
- non-shell command returns `true`.
- empty command, pane missing/helper failure, empty pane id, pre-aborted signal, and in-flight abort return `false`.
- polling remains loop-based with 200ms-capped sleeps.
- Go command surface does not add target-based `display-message` or mutating tmux commands.
