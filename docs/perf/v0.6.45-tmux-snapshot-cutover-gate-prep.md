## Parity Corpus Criteria

The canonical tmux snapshot corpus remains `tests/fixtures/kernel/tmux/snapshotCases.cjs`. v0.6.45 binds the future gate to the current case set:

- empty stdout.
- single row with trailing newline.
- CRLF rows normalize carriage returns.
- malformed rows and rows with too few fields are skipped.
- empty pane id rows are skipped.
- empty labels are retained.
- empty current commands are retained.
- duplicate pane ids keep first order and last values.
- extra tab fields after currentCommand are ignored.
- unicode labels and commands are retained.
- long label and command retained efficiently.
- sentinel-like compact tmux label remains compact and is not treated as full-text leakage.
- mixed corpus canonical snapshot.

A future cutover gate must prove the TypeScript parser, direct Go helper, and adapter-backed Go helper all match this corpus exactly. Any corpus expansion should be deterministic fixture data, not raw logs or terminal captures.

## Fail-Closed Criteria

For explicit cutover modes, parser unavailability must produce an unknown/stale snapshot:

```text
ok:false
status:"unknown"
resultMarker:"stale"
module:"tmuxSnapshotParse"
capability:"tmuxSnapshotParse"
panes:[]
byPaneId:{}
```

The failure must include a compact `cutoverFailureKind` and reason. It must not be a successful empty snapshot and must not call the TypeScript parser fallback callback in explicit cutover failure paths.

Covered failure classes remain:

- missing helper.
- disabled helper.
- unsupported protocol, version, or capability.
- timeout, spawn error, crash, nonzero exit, and empty response.
- malformed JSON and JSON-RPC error.
- incompatible response and unsafe response shape.
- previous helper failure.

Parser failure is not evidence that panes disappeared. It must not clear pane bindings, kill panes, mark workers error, force reconcile, write state, or surface hidden TypeScript live-parser orphan data behind a cutover failure.

The v0.6.45 guard asserts these exact fail-closed criteria:

- TypeScript parser fallback callback is not invoked.
- Failure is not a successful empty snapshot.
- Failure does not clear pane bindings.
- Failure does not kill panes.
- Failure does not mark workers error.
- Failure does not write state.

## No-Leak Criteria

Cutover diagnostics and gate evidence must not include:

- raw stdout.
- raw stderr.
- absolute helper paths, repository paths, or cwd paths.
- raw stack traces.
- raw tmux stdout/stderr logs.
- full mailbox/report bodies.
- worker transcripts.
- terminal raw logs.
- state archives or raw team JSON.
- provider response ids, tool-call ids, secrets, keys, certificates, or raw hosted records.

Allowed evidence is compact and deterministic: case ids, pass/fail status, stable failure kind, module/capability name, package/runtime invariants, and sanitized docs/fixtures/guards.
