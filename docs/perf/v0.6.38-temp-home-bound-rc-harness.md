# v0.6.38 Temp-Home-Bound RC Harness

> Scope: v0.6.38 evidence-only enablement for running representative AgentTeam `/team config` and `agentteam_*` flows against an isolated `PI_AGENTTEAM_HOME`. This is a harness/procedure checkpoint, not a release-readiness approval.

## Result

Result: v0.6.38 provides a temp-home-bound RC harness and operator procedure for the next manual RC slice.

The harness is GO for non-interactive verification of registered `/team config` command handling and representative `agentteam_*` tool flows in an isolated temp home.

The harness is STOP for release/tag/npm/native/default-Go/package/signing/second-platform/fallback-deletion work, production runtime semantics changes, broad Go authority, raw full-text evidence check-in, and claiming real TUI/model/manual RC pass.

No production TypeScript or Go runtime semantics are changed by this enablement.

## Why This Exists

T112 could not safely execute the Slice 5 manual RC checklist because the worker's live `agentteam_*` tools were bound to the active agentteam, not a clean temp `PI_AGENTTEAM_HOME`. Running those live tools would have polluted the current team.

`pi --help` does not expose a direct non-interactive `/team` CLI. The safe enablement path is therefore a script-level harness that loads the extension into a stub Pi API, binds all AgentTeam state paths through `PI_AGENTTEAM_HOME`, and calls the registered command/tool seams directly.

## Automated Harness

Command:

```bash
node scripts/verify-v0638-temp-home-rc-harness.cjs --out /tmp/pi-agentteam-v0638-temp-home-rc-harness-summary.json
```

What the harness does:

- Creates or accepts a temp home only under `/tmp/pi-agentteam-v0638-rc-harness.*`.
- Sets `PI_AGENTTEAM_HOME` before loading/transpiling extension modules, so `state/paths.ts` binds file-backed state to the isolated home.
- Builds a temp CommonJS dist under `/tmp`, with stub Pi/TUI/typebox modules; no build output is written to the repo.
- Registers the real extension through `index.ts` and real `api/commands.ts` / `api/tools.ts` seams.
- Patches tmux operations with in-memory fake panes so spawn exercises AgentTeam state/tool behavior without a real provider/model call.
- Runs `/team config show`, `/team config init`, `/team config validate`, and `/team config migrate --dry-run` through the registered command handler.
- Runs representative `agentteam_create`, `agentteam_spawn`, `agentteam_task`, `agentteam_send`, `agentteam_receive`, `report_done`, `report_blocked`, leader review/close/block/unblock, compact panel model read, legacy `teams/-` absence, and release-governance absence checks.
- Emits sanitized JSON only; the sentinel `V0638_RC_FULL_TEXT_SENTINEL_DO_NOT_LEAK` must never appear in output.
- Cleans up the temp home by default and restores the caller's original `PI_AGENTTEAM_HOME` environment value.

Representative passing output shape:

```json
{
  "ok": true,
  "status": "passed",
  "cleanupResult": "removed",
  "checks": [
    { "id": "worker-receive-boundary", "pass": true },
    { "id": "report-done-report-only", "pass": true },
    { "id": "report-blocked-report-only", "pass": true }
  ]
}
```

The JSON summary is compact evidence only. It must not be checked in if it contains local temp paths or operator-specific metadata; store run output under `/tmp` unless a later task explicitly authorizes a sanitized evidence artifact.

## Isolation Contract

The harness fails closed unless these facts hold:

- `PI_AGENTTEAM_HOME` is non-empty, absolute, under `/tmp/pi-agentteam-v0638-rc-harness.*`, and not under the repo.
- The temp home starts empty unless `--allow-non-empty-home` is explicitly used by a test.
- The original live `PI_AGENTTEAM_HOME` is restored after the run.
- The temp home is removed by default; use `--keep-home` only for local debugging, never for release evidence unless the preserved path is recorded outside the repo.
- No root `package-lock.json`, `npm-shrinkwrap.json`, `go.mod`, `go.sum`, native archive/binary, signing material, generated release manifest, release bundle, or package artifact is required or created by the harness.

## Procedure For Next Manual RC

Use this sequence before attempting the full Slice 5 manual RC again:

1. From repo root, run:
   ```bash
   node scripts/verify-v0638-temp-home-rc-harness.cjs --out /tmp/pi-agentteam-v0638-temp-home-rc-harness-summary.json
   ```
2. Confirm the JSON has `ok: true`, `status: "passed"`, `cleanupResult: "removed"`, and no `errors`.
3. Confirm the summary's `unsupported` list still includes `real-pi-tui-team-panel` and `real-llm-provider-worker-execution`; those remain manual/operator observations, not script proof.
4. For the real manual RC, launch a separate operator pi session with an explicit temp home and extension auto-discovery disabled so the explicit local AgentTeam extension is the only AgentTeam extension loaded:
   ```bash
   export PI_AGENTTEAM_HOME=$(mktemp -d /tmp/pi-agentteam-v0638-rc-smoke.XXXXXX)
   pi --no-extensions --extension ./index.ts --session-dir "$PI_AGENTTEAM_HOME/pi-sessions"
   ```
5. In that isolated pi session, execute the Slice 5 checklist. Use `/team config ...` and `agentteam_*` tools only in that session, not from the live team harness.
6. Record compact pass/fail/blocked notes only: ids, counts, statuses, report ids, task ids, pane ids, and summaries. Do not record raw mailbox/report bodies, worker transcripts, screenshots, state archives, or secrets.
7. Cleanup only the recorded temp path after verifying it starts with `/tmp/pi-agentteam-v0638-rc-smoke.`.

## Coverage And Limits

Covered by the automated harness:

- Temp-home isolation and cleanup safety.
- `/team config show/init/validate/migrate --dry-run` command handling.
- Safe ASCII team creation and unsafe Chinese-only/marker-only rejection.
- Stubbed visible worker spawn through `agentteam_spawn` without model/provider execution.
- Task create/assign/send/receive/report_done/report_blocked/leader review/block/unblock/close flow.
- `agentteam_receive` as the mailbox full-text boundary in a temp-home-bound actor context.
- `agentteam_task action=report` as the TaskReport full-text boundary in a temp-home-bound leader context.
- Non-leader reports stay report-only until explicit leader action.
- Compact panel model can be read without full-body sentinel leakage.
- Legacy `teams/-` absence and release-governance absence checks.

Not covered by the automated harness:

- Real interactive `/team` TUI rendering, flicker, key handling, attached/global panel operator observation, and real terminal paint behavior.
- Real LLM/model/provider worker execution; spawn uses fake tmux panes and simulated worker session binding.
- Real tmux pane lifecycle beyond the in-memory fake tmux adapter used by the script.
- p95 evidence, broad `npm test` green, release readiness, or manual RC pass.

## STOP Gates

STOP if any of these occur:

- The harness would use the live/default `PI_AGENTTEAM_HOME` or a path outside `/tmp/pi-agentteam-v0638-rc-harness.*`.
- The operator procedure would use live `agentteam_*` tools instead of an isolated pi session.
- Raw full `MailboxMessage.text`, `TaskReport.text`, worker transcript, screenshot, state archive, provider output, secret, or local temp home archive would be checked into the repo.
- Any command attempts `git tag`, `git push`, release creation, `npm version`, `npm publish`, native/default-Go/default-resolver/package/signing/second-platform/fallback-deletion work, or production runtime edits.
- Script output contains `V0638_RC_FULL_TEXT_SENTINEL_DO_NOT_LEAK`.
