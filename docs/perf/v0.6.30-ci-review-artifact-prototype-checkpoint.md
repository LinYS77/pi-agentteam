# v0.6.30 CI Review Artifact Prototype Checkpoint

> Scope: final checkpoint for the v0.6.30 CI review artifact prototype. This is GitHub-only review evidence for a single linux-x64-glibc helper artifact path. It does not approve release assets, package install, default Go, default resolver, runtime download, normal-user native availability, `/team readiness`, normal-user UI/tool/runtime diagnostics, or TypeScript fallback deletion.

## Decision

GO for checkpoint evidence only:

- `.github/workflows/go-helper-review-artifact.yml` exists as the only approved review-only GitHub Actions workflow.
- workflow permissions remain minimal: `permissions: contents: read`.
- workflow builds via `node scripts/build-go-helper-artifact.cjs --output-root "$output_root" --ci-review --json` and uploads a review-only GitHub Actions artifact with `actions/upload-artifact@v4`, `if-no-files-found: error`, and `retention-days: 7`.
- `scripts/lib/go-helper-artifact-builder.cjs` and `scripts/build-go-helper-artifact.cjs` can generate `artifact-index.json` in opt-in CI review mode.
- `artifact-index.json` records `reviewOnly: true`, `releaseAsset: false`, `installSource: false`, and `normalUserAvailability: false`.
- first matrix evidence is exactly the required `ubuntu-latest / linux-x64-glibc` row; macOS, Windows, arm64, musl, and other rows remain unsupported/future and are not claimed as normal-user availability.
- workflow verify job uses `actions/download-artifact@v4` and `node scripts/verify-go-helper-artifact.cjs --artifact-root "$artifact_root" --json`.
- `scripts/lib/go-helper-artifact-verifier.cjs` validates `artifact-index.json`, `manifest.json`, `SHA256SUMS`, `provenance.json`, `LICENSE`, `license.json`, placeholder-only `attestation.intoto.jsonl`, safe package-relative paths, hash/size integrity, direct JSON-RPC `health` and `tmuxSnapshotParse` smoke, and explicit `go-packaged-preview` adapter smoke.
- explicit preview smoke proves `tmuxSnapshotParse` uses the helper while `compactReadModelFingerprint` remains TypeScript fallback and does not call the helper.
- Slice 5 reviewer usability hardening covers compact diagnostics for incomplete root/path pairs, unsupported platform/libc, missing executable, version/protocol skew, checksum/integrity mismatch, executable-mode mistakes, metadata basename-only exposure, and no-leak behavior.

## STOP / Non-Approvals

This checkpoint explicitly does not approve:

- GitHub Actions artifacts as release assets, install sources, package artifacts, package-manager install proof, or normal-user native availability proof.
- `npm version`, `npm publish`, package metadata changes, `package.json#files` changes for native artifacts, `optionalDependencies`, package scripts, lifecycle hooks, postinstall, download, or install-time build.
- `go.mod`, `go.sum`, npm lockfiles, or shrinkwrap.
- checked-in native binaries, generated artifacts, generated manifests, checksums, provenance, attestations, tarballs, zips, or artifact bundles.
- GitHub release assets, npm companion packages, or main package native helper inclusion.
- default Go, default resolver, current `go-cutover` behavior changes, TypeScript fallback deletion, or `compactReadModelFingerprint` cutover.
- broad Go authority, runtime artifact download, package-manager install proof, `/team readiness` expansion, normal-user UI diagnostics, model-callable tools, ambient runtime diagnostics, network listeners, tmux execution, repository/state writes, worker lifecycle authority, PlanRun/governance authority, mailbox/report full-text access, or package/release authority.

## Remaining Blockers

Before any normal-user native/default/package claim, the following remain blocked:

- hosted GitHub Actions workflow run evidence has not been executed/observed in this local checkpoint.
- cross-platform matrix evidence beyond `linux-x64-glibc`, including macOS, Windows, arm64, musl, and other libc/architecture rows.
- real package-manager install proof from a clean package install path.
- package metadata ownership, release ownership, npm strategy, companion package strategy, and rollback responsibility.
- default resolver and default Go approval.
- fallback deletion approval plus rollback/default-disable plan.
- package/release security ownership for provenance, attestation, retention, artifact naming, signing policy, and supported-platform commitments.
