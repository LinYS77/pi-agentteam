# v0.6.56 Go inspectPane Facade Cutover

Result: v0.6.56 cuts over the TypeScript `inspectPane(paneId)` facade/default path to the existing Go `workerLifecycle.inspectPane` adapter.

This slice closes the remaining facade fallback gap for the already implemented read-only pane inspection operation:

```text
tmux/core.ts inspectPane(paneId) -> TypeScript kernel adapter -> Go workerLifecycle.inspectPane -> compact PaneInspection facade result
```

## Runtime Behavior

- `tmux/core.ts` `inspectPane(paneId)` now delegates to `createAgentTeamKernelAdapter().inspectWorkerPane(paneId)`.
- The TypeScript `runTmuxNoThrow(['display-message', '-p', '-t', paneId, ...])` fallback for this facade is removed.
- `inspectPane(paneId)` preserves the existing `PaneInspection` facade shape.
- Successful adapter results map to `{ paneId, exists:true, currentCommand, inMode, mode, copyMode }`.
- Failed adapter results map to `{ paneId, exists:false, error }`.
- The Go worker lifecycle adapter remains responsible for the v0.6.53 compact inspection semantics and uses the existing read-only `tmux list-panes -a -F workerLifecycleInspectPaneFormat` path.

## Failure Policy

When the Go helper is missing, unavailable, incompatible, cannot find the pane, or returns a failed worker lifecycle result, the public facade returns the existing compact failure shape with `exists:false`.

Kernel-level diagnostics remain compact on the adapter result and do not leak raw stdout, stderr, cwd, stack traces, state archives, mailbox bodies, report bodies, or worker transcripts.

## Preserved Boundaries

- `targetForPaneId()`, `captureCurrentPaneBinding()`, `paneExists()`, and `resolvePaneBinding()` remain TypeScript `display-message` paths.
- `windowExists()` and `firstPaneInWindow()` remain TypeScript window helper paths.
- `listAgentTeamPanes()` remains delegated to `createAgentTeamKernelAdapter().listAgentTeamPanes()`.
- Mutating worker lifecycle remains TypeScript-owned: no `wakePane`, `syncPaneLabels`, `createTeammatePane`, `killPane`, `clearPaneLabel`, send-keys, label mutation, split-window, new-window, kill-pane, select-pane, or respawn-pane migration.
- State repository, task/report/PlanRun governance, team panel view-model, release/package verification, and package/native ownership remain unmigrated.
- No Go source or native helper rebuild is required for this facade-only cutover.
