## Harness

Focused command:

```bash
PI_AGENTTEAM_PROFILE=1 node scripts/verify-v0639-task-message-report-p95.cjs --out /tmp/pi-agentteam-v0639-task-message-report-p95-latest.json
```

The verifier calls `scripts/lib/v0639-task-message-report-p95-harness.cjs`, which:

- Creates a clean temporary `PI_AGENTTEAM_HOME` under `/tmp/pi-agentteam-v0639-task-message-report-p95.*` and rejects unsafe/non-empty homes by default.
- Registers the real `agentteam_*` tool definitions with file-backed test dependencies and exercises real task/message/report application seams with stubbed pi/tmux/delivery dependencies.
- Times app-owned registered tool execution with mailbox/ref side effects and stubbed delivery/attention boundaries; it excludes LLM/provider time, real worker execution, native pi wake handling after enqueue, real tmux startup, terminal rendering, and operator think time.
- Keeps raw timing JSON under `/tmp` and checks that the sentinel `V0639_TASK_MESSAGE_REPORT_FULL_TEXT_SENTINEL_DO_NOT_LEAK` does not appear in sanitized summary or raw timing output.
- Restores the original `PI_AGENTTEAM_HOME` and `PI_AGENTTEAM_TEST_AUTO_BRIDGE` after the run and removes the temp home unless `--keep-home` is explicitly used.

## Coverage

Covered task/message/report actions:

- `agentteam_task` create/assign/close/block/unblock, plus `report_done` and `report_blocked` report-only paths.
- `agentteam_send` assignment/question/inform through task-bound delivery.
- `agentteam_receive markRead=false` and `agentteam_receive markRead=true` as the mailbox full-text/read boundary.
- Worker-owned `report_done` and `report_blocked` are report-only for worker owners: the worker report does not directly move task status to `done` or `blocked`; leader close/block/unblock changes the task state afterward.
